Detailed Description

Test fixture providing automatic transaction rollback for test isolation.

Purpose: Ensures each test runs in complete isolation by wrapping all database operations in a transaction that is unconditionally rolled back at the end.

Benefits:

  • Zero test data pollution between tests
  • No cleanup code required in tests
  • Tests can run in any order
  • Parallel test execution safe (if using separate connections)

Usage with doctest:

#define DOCTEST_CONFIG_IMPLEMENT_WITH_MAIN
#include <doctest/doctest.h>
#include <muscovite/dba/test_fixture.hpp>
SCENARIO("Patient CRUD operations") {
GIVEN("a live database connection") {
auto pool = ConnectionPool::create(std::getenv("MUSCOVITE_TEST_DB"));
TestTransactionFixture fixture(pool);
auto& txn = fixture.transaction();
WHEN("inserting a new patient") {
PatientRepository repo;
auto patient = repo.insert(txn, create_test_patient());
THEN("the patient has a valid id") {
CHECK(patient.id() != boost::uuids::nil_uuid());
}
}
// Fixture destructor rolls back - patient is NOT persisted
}
}

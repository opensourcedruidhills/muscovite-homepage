Detailed Description

Muscovite gRPC OAuth2 Library

Optional authentication library for gRPC services.

Features:

  • OAuth2 Authorization Code Flow with PKCE (RFC 6749 + RFC 7636)
  • Built-in authentication against muscoviteserver security module
  • External identity provider support (Keycloak, Auth0, Azure AD)
  • Automatic token refresh before expiry
  • Secure token storage (memory + OS keychain)
  • JWT parsing and validation
  • gRPC interceptor for automatic Authorization header injection

Usage:

#include <muscovite/grpc/oauth2/oauth2.hpp>
using namespace muscovite::grpc::oauth2;
// Create configuration for external provider
auto config = OAuth2Config::forKeycloak(
"https://keycloak.example.com/realms/myrealm",
"my-client-id",
"http://localhost:8080/callback"
);
// Create authorization code flow handler
auto flow = std::make_shared<AuthorizationCodeFlow>(config);
// Create gRPC interceptor with oauth2 support
auto interceptor = std::make_shared<OAuth2Interceptor>(flow);
// Start OAuth2 flow - user opens this URL in browser
auto auth_url = interceptor->startOAuth2Flow();
std::cout << "Open this URL: " << auth_url << std::endl;
// After user authenticates and is redirected back with code:
interceptor->completeOAuth2Flow(authorization_code);
// Create gRPC channel with OAuth2 authentication
auto channel = createSecureChannelWithOAuth2(
"grpc.example.com:443",
interceptor
);
// Create service stubs - all RPCs now include Authorization header
MyService::Stub service(channel);
// ... use service ...
// Logout and revoke tokens
interceptor->logout();

Convenience header for the complete OAuth2 authentication library.

Dual-Mode Authentication

The OAuth2Interceptor supports both:

  1. Built-in mode: Username/password against muscoviteserver
    interceptor->authenticateBuiltIn(username, password);
  2. External OAuth2 mode: Any OIDC-compliant identity provider
    auto auth_url = interceptor->startOAuth2Flow();
    // ... user authenticates at provider ...
    interceptor->completeOAuth2Flow(code);

Configuration

Supports multiple identity provider factories:

  • OAuth2Config::forKeycloak(realm_url, client_id, redirect_uri)
  • OAuth2Config::forAuth0(domain, client_id, redirect_uri)
  • OAuth2Config::forAzureAD(tenant_id, client_id, redirect_uri)
  • OAuth2Config::forBuiltIn(auth_endpoint) - Built-in muscoviteserver

Security Features

  • PKCE (Proof Key for Code Exchange) for public clients
  • Token expiration tracking and automatic refresh
  • Secure token storage (access token in memory, refresh token in OS keychain)
  • Token revocation on logout
  • JWT parsing and claims validation
  • Clock skew tolerance for timestamp validation

Thread Safety

OAuth2Interceptor is thread-safe. Multiple threads can call GetMetadata() concurrently. Token refresh is synchronized.

Detailed Description

Generic QAbstractListModel for QML ListViews.

Stores items as QVariantMap and auto-derives role names from the keys of the first inserted item. Generated ViewModels own an instance and call replaceItems() after gRPC list fetches; QML delegates bind via model.roleName syntax.

Incremental updates
  • appendItem() → beginInsertRows/endInsertRows → ListView add-animation
  • removeItem() → beginRemoveRows/endRemoveRows → ListView remove-animation
  • replaceItems() → beginResetModel/endResetModel → full refresh
Role derivation
Roles are derived from the keys of the first item in the model. Qt::UserRole + 1 is mapped to the first key, +2 to the second, etc. Qt::DisplayRole is aliased to the "display" key (or first key if absent).
Example (generated code)
// In ViewModel constructor:
itemModel_ = new EntityListModel(this);
// After gRPC fetch:
QVariantList items;
for (const auto& e : response.items()) {
QVariantMap row;
row["id"] = QString::fromStdString(e.id().value());
row["name"] = QString::fromStdString(e.name());
items.append(row);
}
itemModel_->replaceItems(items);

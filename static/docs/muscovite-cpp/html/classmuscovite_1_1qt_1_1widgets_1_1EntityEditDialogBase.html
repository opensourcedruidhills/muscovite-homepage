Detailed Description

Base class for entity edit dialogs.

Provides a modal dialog with:

  • QFormLayout for field editors (generated per-entity)
  • QDialogButtonBox with Save/Cancel buttons
  • Validation framework with field-level error display
  • Loading state during gRPC save operations
  • Support for both create (new entity) and edit (existing entity) modes

Subclasses (generated per-entity) must implement:

  • setupFields() - add field editors to the form layout
  • populateFromEntity(const QString& id) - load entity data for editing
  • collectToRequest() - gather field values into gRPC request
  • saveEntity() - call gRPC Create or Update method

Usage: auto* dialog = new ChemicalElementEditDialog(channel, authMgr, this); dialog->setEditMode(entityId); // or dialog->setCreateMode(); if (dialog->exec() == QDialog::Accepted) { listWidget->refresh(); }

Handle validation errors from a synchronous gRPC call.

Extracts validation metadata from the ClientContext's trailing metadata and shows any validation errors in the dialog's validation display.

Parameters
ctxThe ClientContext after the gRPC call (must be called after stub->Method())
statusThe gRPC status returned from the call
Returns
true if no validation error (call succeeded or failed without validation), false if a validation error was extracted and shown

Example usage in generated code:

auto status = stub->CreateEntity(ctx.get(), request, &response);

Detailed Description

Transport-agnostic search indexer.

Processes domain events by delegating to registered DocumentMappers. The indexer maintains a registry of mappers (one per entity type) and routes events based on aggregate_type and event_type.

Thread-safety: process_event() is safe to call from multiple threads. Mapper registration (register_mapper()) must happen before processing starts.

Invariant
Each aggregate_type has at most one registered mapper
Metrics counters are monotonically increasing

Usage:

OpenSearchClient client(config);
SearchIndexer indexer(client);
auto mapper = std::make_shared<PatientDocumentMapper>(query_repo);
indexer.register_mapper(mapper);
indexer.ensure_indexes(); // Create indexes with mappings
// In NATS handler (generated code):
indexer.process_event(SearchEvent{
.aggregate_type = "Patient",
.aggregate_id = "uuid-string",
.event_type = "PatientCreated",
.payload = event_json
});

◆ process_event()

auto muscovite::search::SearchIndexer::process_event ( const SearchEvent & event) -> IndexResult

Process a single domain event.

Routes the event to the appropriate DocumentMapper based on aggregate_type. Events ending in "Deleted" trigger document deletion. All other events trigger fetch_and_map() + index_document().

Thread-safe: multiple events can be processed concurrently.

Parameters
eventThe domain event to process
Returns
IndexResult indicating what action was taken

Register a document mapper for an entity type.

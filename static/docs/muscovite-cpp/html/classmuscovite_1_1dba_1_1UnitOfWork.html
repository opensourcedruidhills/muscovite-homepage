Detailed Description

Unit of Work pattern implementation.

Provides a single transaction context for business operations with automatic domain event collection and persistence.

Key Features:

  • Single transaction for all repository operations
  • Collects domain events during the operation
  • Persists all events atomically on commit()
  • Events are discarded on rollback()

Usage:

void PatientService::create(const CreatePatientRequest& req) {
UnitOfWork uow(pool_, "basedata"); // schema for domain_events table
auto& txn = uow.transaction();
auto patient = patient_repo_.insert(txn, req); // Repo stays pure CRUD
uow.add_event(DomainEvent::create(
"PatientCreated",
patient.id(),
"Patient",
nlohmann::json{{"mrn", patient.mrn()}}
));
uow.commit(); // Events persisted atomically
}

Detailed Description

Client-side gRPC error classification for generated stubs (FTR-233).

Provides is_retryable(), is_permanent(), and is_deadline_exceeded() utility functions for generated gRPC client code. Classification follows standard gRPC retry semantics:

Status Code Retryable Permanent
ABORTED ✓
UNAVAILABLE ✓
DEADLINE_EXCEEDED ✓ (*)
RESOURCE_EXHAUSTED ✓
NOT_FOUND ✓
ALREADY_EXISTS ✓
INVALID_ARGUMENT ✓
PERMISSION_DENIED ✓
UNAUTHENTICATED ✓
UNIMPLEMENTED ✓
FAILED_PRECONDITION ✓

(*) DEADLINE_EXCEEDED is retryable only if there is enough remaining budget. Use is_deadline_exceeded() to distinguish this case.

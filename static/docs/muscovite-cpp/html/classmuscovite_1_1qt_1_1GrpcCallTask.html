Detailed Description

A QObject task that runs a blocking gRPC unary call on a worker thread.

Compatible with QtTaskTree::QDefaultTaskAdapter<GrpcCallTask>. Configure via setCallFunction() in the QCustomTask setup handler. The call function runs on a worker thread (via QtConcurrent::run) and the done(DoneResult) signal is emitted on the main thread when complete.

Note
The call function should capture the stub, request, and a shared_ptr to the response. GrpcCallTask itself is non-template — type safety comes from the captured lambda.
Usage (generated code pattern):
auto resp = std::make_shared<GetPatientResponse>();
GrpcUnaryTask task(
[stub, req, resp](GrpcCallTask& t) {
t.setCallFunction([=]() {
grpc::ClientContext ctx;
ctx.set_deadline(
std::chrono::system_clock::now() + std::chrono::seconds{10});
return stub->GetPatient(&ctx, req, resp.get());
});
return QtTaskTree::SetupResult::Continue;
},
[resp](const GrpcCallTask& t, QtTaskTree::DoneWith dw) {
if (dw == QtTaskTree::DoneWith::Success)
qDebug() << QString::fromStdString(resp->family_name());
return QtTaskTree::DoneResult::Success;
});

Cancellation
TaskTree handles cancellation by destroying the GrpcCallTask. If the worker thread has already started the gRPC call, it will complete normally but the done handler will not be invoked (the tree has moved on). For explicit cancellation of in-flight RPCs, capture a shared grpc::ClientContext and call TryCancel() from the main thread.

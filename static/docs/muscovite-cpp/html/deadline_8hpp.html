Clamp an incoming gRPC deadline to a per-aggregate maximum.

When an aggregate declares GRPC_DEADLINE "Ns", the generated service code calls this function instead of using context->deadline() directly. The returned time_point is the earlier of the client-supplied deadline and now + max_duration, preventing a single slow client from holding a database connection beyond the aggregate's SLA.

Parameters
contextThe gRPC ServerContext carrying the client deadline
max_durationMaximum allowed duration for this aggregate
Returns
The earlier of (now + max_duration) and the client's deadline

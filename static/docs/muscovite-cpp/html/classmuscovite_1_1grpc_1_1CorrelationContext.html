Detailed Description

Thread-local holder for request-scoped correlation ID (UUIDv4).

Every incoming gRPC request receives a correlation ID — either extracted from the x-correlation-id metadata header or generated fresh. The ID follows the request through CommandService → Repository → domain events → NATS outbox → log lines.

Thread-local storage is safe for gRPC's one-thread-per-request model.

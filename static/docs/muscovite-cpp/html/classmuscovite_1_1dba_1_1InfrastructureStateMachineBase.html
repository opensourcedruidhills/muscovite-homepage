Detailed Description

template<typename BackoffPolicy>
class muscovite::dba::InfrastructureStateMachineBase< BackoffPolicy >

Three-state circuit breaker: Closed → Open → Half-Open.

This is NOT a domain state machine. It tracks infrastructure health for plugin calls (gRPC, QtRemoteObjects). The BackoffPolicy template parameter controls the cool-down schedule between Open → Half-Open probe attempts.

State transitions:

  • Closed: normal operation. Failures increment counter.
  • Open: all calls rejected. After backoff delay, transitions to Half-Open.
  • Half-Open: one probe call allowed. Success → Closed, Failure → Open.
Template Parameters
BackoffPolicyMust satisfy: next_delay(int) → chrono::milliseconds, reset().
See also
ADR 20260223-circuit-breaker-as-infrastructure-state-machine
Note
This class is NOT related to muscovite::qt::validation::EntityDialogStateMachine. That hierarchy models domain entity lifecycles; this models infrastructure health.

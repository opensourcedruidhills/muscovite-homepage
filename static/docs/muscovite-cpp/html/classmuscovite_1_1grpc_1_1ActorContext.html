Detailed Description

Thread-local holder for the authenticated caller's principal UUID string.

The auth interceptor places the caller's principal UUID in gRPC metadata under the x-principal-id key after JWT validation. Generated service handlers extract this value and store it here via ActorContext::set() before calling into the DBA layer. Generated command repositories read it via ActorContext::current_as_string_opt() to populate created_by/updated_by.

Thread-local storage is safe for gRPC's one-thread-per-request model.

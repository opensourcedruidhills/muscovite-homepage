Detailed Description

Connection routing for read replicas in CQRS architecture.

This module provides connection routing for CQRS read/write split:

  • CommandService operations → Primary database (writes)
  • QueryService operations → Read replica (if configured) or primary (fallback)

Usage in generated QueryService:

class PatientQueryService {
public:
PatientQueryService(ReadReplicaRouter& router)
: router_(router) {}
std::optional<Patient> find_by_id(const uuid& id) {
auto conn = router_.acquire_read(); // Uses replica if available
// ... query logic
}
private:
ReadReplicaRouter& router_;
};

Configuration in server.ini:

[database]
primary = postgresql://postgres:postgres@localhost:5432/mydb
[read_replica]
enabled = true
connection_string = postgresql://postgres:postgres@replica:5432/mydb
fallback_to_primary = true
health_check_interval = 30

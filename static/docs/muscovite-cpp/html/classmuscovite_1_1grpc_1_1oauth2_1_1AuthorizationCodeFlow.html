Detailed Description

OAuth2 Authorization Code Flow with PKCE.

Implements RFC 6749 (OAuth 2.0 Authorization Framework) with RFC 7636 (Proof Key for Public Clients) extension.

Flow:

  1. Client generates PKCE parameters and opens authorization request
  2. User authenticates and grants consent at identity provider
  3. Identity provider redirects to redirect_uri with authorization code
  4. Client exchanges code + PKCE verifier for tokens at token endpoint
  5. Client uses access token in Authorization header of API requests
  6. When token expires, client uses refresh token to get new access token

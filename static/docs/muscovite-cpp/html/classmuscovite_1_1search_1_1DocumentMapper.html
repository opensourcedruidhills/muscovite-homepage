Detailed Description

Abstract interface for mapping domain entities to OpenSearch documents.

Generated implementations (one per searchable entity) provide:

  • Index name derivation from bounded context + entity name
  • Index mapping (field types, analyzers, keyword sub-fields)
  • Entity fetch + JSON serialization for document body
Invariant
Each DocumentMapper handles exactly one entity type
index_name() returns a stable, deterministic name

Example generated implementation:

class PatientDocumentMapper : public DocumentMapper {
PatientQueryRepository& repo_;
public:
std::string index_name() const override { return "patient_patient"; }
std::string entity_type() const override { return "Patient"; }
nlohmann::json index_mapping() const override { ... }
std::optional<nlohmann::json> fetch_and_map(const std::string& id) override {
auto patient = repo_.find_by_id(parse_uuid(id));
if (!patient) return std::nullopt;
return nlohmann::json{{"mrn", patient->mrn}, {"family_name", patient->family_name}};
}
};

Fetch current entity state and map to search document.

Called on *Created and *Updated events. The mapper should:

  1. Fetch the entity from its QueryRepository by ID
  2. Map all @searchable fields to a flat JSON document
  3. Return nullopt if entity not found (deleted between event and processing)

FGDC/USGS Geologic Map Symbolization Standard

FGDC/USGS Geologic Map Symbolization Standard

  • Status: accepted
  • Deciders: Domain modeling team
  • Date: 2026-01-07
  • Tags: mapping, cartography, FGDC, USGS, symbols

Context and Problem Statement

Geologic maps require standardized symbols for features like faults, folds, contacts, attitudes, and various point/line/area features. The USGS/FGDC standard provides authoritative symbology that should be modeled in our mapping context.

Decision Drivers

  • FGDC Digital Cartographic Standard for Geologic Map Symbolization is the US national standard
  • Standard applies to all Federal Government geologic map publications
  • ArcGIS Pro and QGIS styles available for implementation
  • Need comprehensive symbol library for map generation
  • Symbols must be scale-independent

Considered Options

  • Minimal symbol set for basic needs
  • Full FGDC standard implementation with all categories
  • Extensible symbol library with FGDC as base

Decision Outcome

Chosen option: "Extensible symbol library with FGDC as base", because:

  1. FGDC standard is authoritative and well-documented
  2. Extensible design allows custom symbols while maintaining standards compliance
  3. SVG-based symbols enable web and print output
  4. Symbol categories match geologic feature types

FGDC Symbol Categories

The standard organizes symbols into major sections:

Section Description Example Symbols
1 Contacts, key beds, dikes Contacts (solid, dashed, inferred, concealed)
2 Faults Normal, reverse, thrust, strike-slip, detachment
3 Folds Anticline, syncline, monocline, overturned
4 Bedding Horizontal, inclined, vertical, overturned
5 Lineation Mineral, intersection, crenulation
6 Foliation Cleavage, schistosity, gneissic banding
7 Joints Systematic, non-systematic, sheeting
8 Veins Quartz, carbonate, mineralized
9 Unconformities Angular, disconformity, nonconformity
10 Glacial features Moraines, drumlins, eskers, striations
11 Fluvial features Terraces, channels, alluvial fans
12 Coastal features Beaches, dunes, cliffs, shorelines
13 Mass movement Landslides, debris flows, rock falls
14 Volcanic features Craters, vents, lava flows, pyroclastics
15 Karst features Sinkholes, caves, springs
16 Mining/resources Mines, prospects, quarries, wells
17 Paleontology Fossil localities by type
18 Sample localities Rock samples, soil samples, water samples
19 Area patterns Lithology patterns, surficial deposit patterns
20 Map elements Scale bars, north arrows, legends

Symbols to Add

-- Section 9: Unconformity symbols
ENTITY UnconformitySymbol IDENTIFIED_BY id (
    id                    UUID         NOT NULL,
    unconformity_type     VARCHAR(50)  NOT NULL,
    svg_ornament          TEXT,
    spacing_mm            DECIMAL,
    description           TEXT
)
CHECK type_valid: unconformity_type IN (
    'angular_unconformity', 'disconformity', 'nonconformity', 
    'paraconformity', 'buttress_unconformity'
)
RELATIONSHIPS {
    symbol -> MapSymbol REQUIRED ONE;
}

-- Section 10: Glacial feature symbols
ENTITY GlacialSymbol IDENTIFIED_BY id (
    id                    UUID         NOT NULL,
    feature_type          VARCHAR(50)  NOT NULL,
    svg_path              TEXT         NOT NULL,
    decoration_style      VARCHAR(30),
    description           TEXT
)
CHECK type_valid: feature_type IN (
    'moraine_crest', 'drumlin', 'esker', 'kame', 'kettle',
    'glacial_striation', 'roche_moutonnee', 'cirque', 'arete',
    'horn', 'hanging_valley', 'u_shaped_valley', 'ice_margin'
)
RELATIONSHIPS {
    symbol -> MapSymbol REQUIRED ONE;
}

-- Section 13: Mass movement symbols
ENTITY MassMovementSymbol IDENTIFIED_BY id (
    id                    UUID         NOT NULL,
    movement_type         VARCHAR(50)  NOT NULL,
    svg_path              TEXT,
    crown_symbol          TEXT,
    toe_symbol            TEXT,
    direction_indicator   BOOLEAN      NOT NULL DEFAULT true,
    description           TEXT
)
CHECK type_valid: movement_type IN (
    'landslide', 'rockslide', 'debris_flow', 'earth_flow',
    'rock_fall', 'slump', 'creep', 'solifluction'
)
RELATIONSHIPS {
    symbol -> MapSymbol REQUIRED ONE;
}

-- Section 14: Volcanic feature symbols
ENTITY VolcanicSymbol IDENTIFIED_BY id (
    id                    UUID         NOT NULL,
    feature_type          VARCHAR(50)  NOT NULL,
    svg_path              TEXT         NOT NULL,
    default_size_mm       DECIMAL,
    description           TEXT
)
CHECK type_valid: feature_type IN (
    'crater', 'caldera', 'vent', 'fumarole', 'hot_spring',
    'lava_flow_front', 'lava_tube', 'spatter_cone', 'cinder_cone',
    'maar', 'tuff_ring', 'volcanic_dome'
)
RELATIONSHIPS {
    symbol -> MapSymbol REQUIRED ONE;
}

-- Section 16: Resource symbols
ENTITY ResourceSymbol IDENTIFIED_BY id (
    id                    UUID         NOT NULL,
    resource_type         VARCHAR(50)  NOT NULL,
    commodity             VARCHAR(50),
    svg_path              TEXT         NOT NULL,
    status                VARCHAR(30),
    description           TEXT
)
CHECK type_valid: resource_type IN (
    'mine', 'prospect', 'quarry', 'pit', 'adit', 'shaft',
    'oil_well', 'gas_well', 'water_well', 'geothermal_well',
    'mineral_occurrence', 'placer'
)
CHECK status_valid: status IN ('active', 'inactive', 'abandoned', 'reclaimed') OR status IS NULL
RELATIONSHIPS {
    symbol -> MapSymbol REQUIRED ONE;
}

-- Section 17: Paleontology symbols
ENTITY PaleontologySymbol IDENTIFIED_BY id (
    id                    UUID         NOT NULL,
    fossil_type           VARCHAR(50)  NOT NULL,
    svg_path              TEXT         NOT NULL,
    default_size_mm       DECIMAL,
    description           TEXT
)
CHECK type_valid: fossil_type IN (
    'vertebrate', 'invertebrate', 'plant', 'trace_fossil',
    'microfossil', 'palynomorph', 'conodont', 'fish',
    'mammal', 'dinosaur', 'marine_reptile'
)
RELATIONSHIPS {
    symbol -> MapSymbol REQUIRED ONE;
}

Positive Consequences

  • Comprehensive symbol library matching FGDC standard
  • Extensible for custom symbols
  • SVG-based for modern web/print output
  • Enables standards-compliant map production

Negative Consequences

  • Large number of symbol entities
  • SVG definitions require graphic design expertise
  • Maintaining synchronization with FGDC updates

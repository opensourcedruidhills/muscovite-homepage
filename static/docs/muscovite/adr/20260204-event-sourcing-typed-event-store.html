Event Sourcing with Typed Event Store (Option B)

Event Sourcing with Typed Event Store (Option B)

  • Status: accepted
  • Date: 2026-02-04
  • Deciders: User (jolo)
  • Tags: event-sourcing, cqrs, persistence, aggregates

Context and Problem Statement

Muscovite needs event sourcing capability for aggregates that require:

  • Complete audit trail of state changes
  • Ability to reconstruct state from events
  • Temporal queries (state at point in time)
  • Event replay for projections

Event sourcing is distinct from:

  • Domain Events (outbox): Cross-process messaging, side effects, published to NATS
  • Audit Logs: Compliance records of who-did-what-when

Decision Drivers

  • Type safety across all layers (no JSON/JSONB)
  • Opt-in per aggregate (not all need event sourcing complexity)
  • Backward compatible with existing DSL
  • Consistent with existing generator architecture
  • Per-event snapshots for fast state loading

Considered Options

  1. Option A: Single events table with JSONB payload
  2. Option B: Strongly typed per-event payload tables (no JSON)
  3. Option C: Per-context events with Protobuf BYTEA
  4. Option D: Hybrid stream-based (EventStoreDB-inspired)

Decision Outcome

Chosen option: Option B — Strongly Typed Event Store

Every event type gets its own payload table with proper typed columns. This maintains full type safety and aligns with the "no JSON" rule.

Storage Strategy

  • Dual-write: Events + snapshot + entity table (for queries)
  • Snapshots: Updated on every event (no replay needed for reads)
  • Event evolution: Upcasters transform old events on load

DSL Syntax

Opt-in Event Sourcing

-- Standard aggregate (unchanged)
AGGREGATE Patient @uuid "..." ROOT Patient
DOC "Regular aggregate";

-- Event-sourced aggregate (new)
AGGREGATE Patient @uuid "..." ROOT Patient
    EVENT_SOURCED
DOC "Aggregate with event sourcing";

Sourced Events

SOURCED_EVENT PatientRegistered @uuid "..." (
    patient_id    UUID NOT NULL @uuid "...",
    mrn           TEXT NOT NULL @uuid "...",
    given_name    TEXT NOT NULL @uuid "...",
    family_name   TEXT NOT NULL @uuid "..."
)
APPLIES_TO Patient
DOC "Patient was registered";

SOURCED_EVENT PatientNameChanged @uuid "..." (
    patient_id    UUID NOT NULL @uuid "...",
    given_name    TEXT NOT NULL @uuid "...",
    family_name   TEXT NOT NULL @uuid "..."
)
APPLIES_TO Patient
DOC "Patient name was changed";

Distinction from Domain Events

Concept Keyword Purpose Storage
Domain Event EVENT Messaging, outbox {context}.domain_events (transient)
Sourced Event SOURCED_EVENT State reconstruction event_store.* (permanent)

Schema Design

Event Envelope Table

CREATE SCHEMA event_store;

CREATE TABLE event_store.events (
    id                BIGSERIAL PRIMARY KEY,
    aggregate_type    TEXT NOT NULL,
    aggregate_id      UUID NOT NULL,
    version           INTEGER NOT NULL,
    event_type        TEXT NOT NULL,
    occurred_at       TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    correlation_id    UUID,
    causation_id      BIGINT REFERENCES event_store.events(id),
    
    CONSTRAINT uq_aggregate_version UNIQUE (aggregate_type, aggregate_id, version)
);

CREATE INDEX idx_events_aggregate ON event_store.events(aggregate_type, aggregate_id);
CREATE INDEX idx_events_type ON event_store.events(event_type);

Per-Event Payload Tables

CREATE TABLE event_store.patient_registered (
    event_id      BIGINT PRIMARY KEY REFERENCES event_store.events(id),
    patient_id    UUID NOT NULL,
    mrn           TEXT NOT NULL,
    given_name    TEXT NOT NULL,
    family_name   TEXT NOT NULL
);

CREATE TABLE event_store.patient_name_changed (
    event_id      BIGINT PRIMARY KEY REFERENCES event_store.events(id),
    patient_id    UUID NOT NULL,
    given_name    TEXT NOT NULL,
    family_name   TEXT NOT NULL
);

Snapshot Table (Per Aggregate Type)

CREATE TABLE event_store.patient_snapshot (
    aggregate_id   UUID PRIMARY KEY,
    version        INTEGER NOT NULL,
    mrn            TEXT NOT NULL,
    given_name     TEXT NOT NULL,
    family_name    TEXT NOT NULL,
    -- All aggregate fields...
    updated_at     TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

C++ Generated Code

namespace patient::event_store {

// Event structs (generated from SOURCED_EVENT)
struct PatientRegistered {
    boost::uuids::uuid patient_id;
    std::string mrn;
    std::string given_name;
    std::string family_name;
};

// Event variant
using PatientEvent = std::variant<
    PatientRegistered,
    PatientNameChanged,
    PatientDeactivated
>;

// Event store with dual-write
class PatientEventStore {
public:
    explicit PatientEventStore(pqxx::connection& conn);
    
    // Append event (updates snapshot + entity table in single transaction)
    auto append(const boost::uuids::uuid& aggregate_id,
                int64_t expected_version,
                const PatientEvent& event) -> std::expected<int64_t, EventStoreError>;
    
    // Load from snapshot
    auto load_snapshot(const boost::uuids::uuid& aggregate_id) 
        -> std::expected<PatientSnapshot, EventStoreError>;
    
    // Load events (for debugging, projections)
    auto load_events(const boost::uuids::uuid& aggregate_id) 
        -> std::vector<PatientEvent>;
};

// Aggregate with apply methods
class Patient {
public:
    static auto from_snapshot(const PatientSnapshot& snapshot) -> Patient;
    
    // Commands return events
    auto register_patient(const RegisterPatientCommand& cmd) 
        -> std::expected<PatientRegistered, DomainError>;
    
private:
    void apply(const PatientRegistered& e);
    void apply(const PatientNameChanged& e);
};

}

Event Evolution

Adding Optional Field

Old events load with NULL for new optional fields. No upcaster needed.

ALTER TABLE event_store.patient_registered ADD COLUMN birth_date DATE;

Adding Mandatory Field

Requires backfill migration and upcaster:

ALTER TABLE event_store.patient_registered 
    ADD COLUMN preferred_language TEXT NOT NULL DEFAULT 'en';

UPDATE event_store.patient_registered 
    SET preferred_language = 'en' 
    WHERE preferred_language IS NULL;

C++ upcaster:

PatientRegistered upcaster_v1_to_v2(const PatientRegistered_v1& old) {
    return PatientRegistered{
        .patient_id = old.patient_id,
        .mrn = old.mrn,
        .preferred_language = "en"  // Default for old events
    };
}

Validation Rules

  1. SOURCED_EVENT must have APPLIES_TO clause
  2. APPLIES_TO target must be an EVENT_SOURCED aggregate
  3. EVENT_SOURCED aggregates must have at least one SOURCED_EVENT
  4. First field of sourced event should reference aggregate root ID

Consequences

Good

  • Full type safety (no JSON)
  • Schema-validated event payloads
  • Efficient queries on event fields
  • Opt-in complexity only where needed
  • Backward compatible

Bad

  • One table per event type (many tables)
  • Schema migrations for event evolution
  • More complex than JSONB approach

Neutral

  • Dual-write maintains query compatibility
  • Snapshots eliminate replay overhead

Implementation Phases

  1. Parser: Add SOURCED_EVENT, APPLIES_TO, EVENT_SOURCED modifier
  2. Model: Add sourced-event struct, update aggregate with event-sourced?
  3. PostgreSQL: Generate event_store schema with typed tables
  4. C++: Generate EventStore class and aggregate apply methods
  5. Tests: Event store integration tests

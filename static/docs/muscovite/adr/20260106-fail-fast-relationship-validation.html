Terminate Generation on Any Invalid Relationship Reference

Terminate Generation on Any Invalid Relationship Reference

  • Status: accepted
  • Deciders: Project maintainers
  • Date: 2026-01-06
  • Tags: dsl, validation, relationships, fail-fast

Context and Problem Statement

When a RELATIONSHIPS block references a non-existent entity, generators previously handled this silently:

  • SQL generator skipped the FK constraint
  • API generator omitted the parameter
  • Qt generator fell back to inference (sometimes wrong)
  • pgTAP generator skipped FK tests

This "silent skip" pattern caused subtle bugs that only surfaced at runtime:

  • Missing FK constraints allowed orphan records
  • Generated forms lacked expected ComboBoxes
  • API clients got schema mismatches

The pattern appeared in 17+ locations across generators, always using:

(define target (hash-ref registry target-name #f))
(when target  ;; Silently skips if not found!
  ...)

Decision Drivers

  • Fail fast with clear, actionable error messages
  • No partial/broken output from generation
  • Surface DDD model errors during development, not runtime
  • Full context in errors (Context.Aggregate.Entity.Relationship → Target)
  • Consistent validation across all relationship types

Considered Options

  • Keep silent skip (current behavior)
  • Warn but continue generating
  • Terminate on REQUIRED relationships only, warn on OPTIONAL
  • Terminate on ALL invalid relationship references

Decision Outcome

Chosen option: "Terminate on ALL invalid relationship references", because partial output is worse than no output, and both REQUIRED and OPTIONAL relationships indicate semantic intent that must be honored.

Positive Consequences

  • No more silent failures in generators
  • Clear error messages with full location path
  • Errors caught during development, not production
  • Forces DDD model completeness before generation
  • Easier debugging (one clear failure point)

Negative Consequences

  • Stricter validation may block generation during iterative development
  • Must fix all relationship targets before seeing any output
  • Requires all referenced contexts to be included in project.ddd

Implementation

Validation Function

Add to src/dslgen/validate.rkt:

(define (validate-relationship-targets all-contexts)
  "Validate all relationship targets exist. Returns list of validation-issue."
  (define all-entity-names (collect-all-entity-names all-contexts))
  (define issues '())
  
  (for ([ctx all-contexts])
    (define ctx-name (bounded-context-name ctx))
    (for ([agg (bounded-context-aggregates ctx)])
      (define agg-name (entity-name (aggregate-root agg)))
      (for ([ent (cons (aggregate-root agg) (aggregate-entities agg))])
        (define ent-name (entity-name ent))
        (define meta (entity-metadata ent))
        (when meta
          (for ([rel (ent-metadata-relationships meta)])
            (define target (relationship-target-entity rel))
            (unless (set-member? all-entity-names target)
              (set! issues
                (cons (validation-issue
                       'error 'reference
                       (format "~a.~a.~a.~a" ctx-name agg-name ent-name 
                               (relationship-name rel))
                       (format "Target entity '~a' not found in any loaded context"
                               target))
                      issues))))))))
  issues)

Integration Point

In src/dslgen/generate.rkt, before calling any generator:

;; Pre-generation validation
(define rel-issues (validate-relationship-targets all-contexts))
(define dep-issues (validate-entity-dependency-order all-contexts))
(define all-issues (append rel-issues dep-issues))

(when (> (length all-issues) 0)
  (displayln "")
  (displayln "=== VALIDATION ERRORS ===")
  (for ([issue all-issues])
    (display-validation-issue issue))
  (displayln "")
  (displayln (format "Generation terminated: ~a errors detected." (length all-issues)))
  (displayln "Hint: Ensure target entities are declared and their context is included in project.ddd")
  (exit 1))

(printf "Validation passed: ~a entities, ~a relationships~n" 
        entity-count relationship-count)

AI Agent Guidance

When This Decision Applies

  • Running code generation (racket generate.rkt --project ...)
  • Adding new relationships to DDD files
  • Adding new contexts to project.ddd
  • Debugging "entity not found" errors

When This Decision Does NOT Apply

  • Runtime database operations (FK constraints handle this)
  • Application-level validation
  • User input validation in generated clients

Enforcement

  • validate-relationship-targets runs before any generator
  • Non-zero exit code on validation failure
  • Full path in error messages for easy location

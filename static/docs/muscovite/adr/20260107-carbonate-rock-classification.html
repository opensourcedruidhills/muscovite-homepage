Carbonate Rock Classification Systems (Dunham, Folk)

Carbonate Rock Classification Systems (Dunham, Folk)

  • Status: accepted
  • Deciders: Domain modeling team
  • Date: 2026-01-07
  • Tags: petrology, sedimentology, carbonates, classification

Context and Problem Statement

Carbonate rocks (limestones, dolostones) require specialized classification systems different from clastic sedimentary rocks. Two main systems exist: Dunham (depositional texture based) and Folk (allochems and matrix based).

Decision Drivers

  • Dunham classification is most widely used (89% of workers per Lokier & Al Junaibi 2016)
  • Folk classification provides more detailed petrographic information
  • Both systems needed for comprehensive carbonate petrology
  • Classifications determine rock name from microscopic observation

Decision Outcome

Implement both Dunham and Folk classifications as lookup entities, with Dunham as primary.

Dunham Classification (1962, modified by Embry & Klovan 1971)

Based on depositional texture:

  • Mud-supported vs grain-supported fabric
  • Presence/absence of mud matrix
  • Evidence of organic binding
ENTITY DunhamClass IDENTIFIED_BY id (
    id                    UUID        NOT NULL,
    class_name            VARCHAR(50) NOT NULL @unique,
    mud_supported         BOOLEAN,
    grain_supported       BOOLEAN,
    organic_bound         BOOLEAN,
    mud_content           VARCHAR(30),
    grain_content         VARCHAR(30),
    grain_size_2mm_plus   VARCHAR(30),
    binding_type          VARCHAR(30),
    description           TEXT
)

Folk Classification (1959, 1962)

Based on allochem types and matrix:

Allochems (transported grains):

  • Oo = Ooids
  • Bio = Bioclasts (fossils)
  • Pel = Peloids (pellets)
  • Intra = Intraclasts

Matrix suffixes:

  • -sparite = Sparry calcite cement (>10μm crystals)
  • -micrite = Micritic mud matrix (<4μm crystals)
ENTITY FolkCarbonateClass IDENTIFIED_BY id (
    id                    UUID        NOT NULL,
    class_name            VARCHAR(50) NOT NULL @unique,
    primary_allochem      VARCHAR(30),
    secondary_allochem    VARCHAR(30),
    matrix_type           VARCHAR(30) NOT NULL,
    folk_code             VARCHAR(20) NOT NULL @unique,
    description           TEXT
)
CHECK matrix_valid: matrix_type IN ('sparite', 'micrite', 'dismicrite')
CHECK allochem_valid: primary_allochem IN ('oo', 'bio', 'pel', 'intra') OR primary_allochem IS NULL

Folk Sandstone Classification

Folk also developed a sandstone classification based on QFR (Quartz-Feldspar-Rock fragments):

ENTITY FolkSandstoneClass IDENTIFIED_BY id (
    id                    UUID        NOT NULL,
    class_name            VARCHAR(50) NOT NULL @unique,
    q_percent_min         DECIMAL,
    q_percent_max         DECIMAL,
    f_percent_min         DECIMAL,
    f_percent_max         DECIMAL,
    r_percent_min         DECIMAL,
    r_percent_max         DECIMAL,
    matrix_content        VARCHAR(30),
    textural_maturity     VARCHAR(30),
    description           TEXT
)
CHECK maturity_valid: textural_maturity IN ('immature', 'submature', 'mature', 'supermature') OR textural_maturity IS NULL

Positive Consequences

  • Both major carbonate classification systems supported
  • Lookup tables enable consistent classification
  • Folk sandstone classification adds sedimentary rock detail
  • Clear criteria for each class enable automated classification suggestions

Negative Consequences

  • Overlap between Dunham and Folk may cause confusion
  • Petrographic skill required to apply correctly
  • Some intermediate textures difficult to classify

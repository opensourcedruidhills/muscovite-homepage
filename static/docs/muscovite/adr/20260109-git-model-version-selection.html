Git-Based Model Version Selection

Git-Based Model Version Selection

  • Status: accepted
  • Deciders: Project maintainers
  • Date: 2026-01-09
  • Tags: git, versioning, diff, schema-evolution, libgit2, c++
  • Related:
    • Stable UUID Identity (ADR 2 - provides UUIDs)
    • Semantic Model Delta (ADR 3 - consumes output)
    • FHIR Health Sample Domain (test domain)

Technical Story: Provide a CLI tool that extracts DDD model versions from git history, outputting data suitable for Racket semantic diff consumption.

Context and Problem Statement

The semantic diff engine (ADR 0003) requires two IR versions to compare. The question is: how do we obtain those versions?

Git provides:

  • Version snapshots of .ddd files
  • Commit history, branching, merges
  • Stable version identifiers (commit hashes)
  • Author, timestamp, message metadata

Git cannot provide:

  • Semantic understanding of the DSL
  • Rename/move detection at semantic level (needs UUIDs)
  • Migration operation inference

Therefore, Git is a version selection tool, not a diff mechanism. We need a clean boundary:

┌─────────────────────────────────────────────────────────────────────┐
│                         Git Domain                                   │
│  ┌─────────────┐    ┌─────────────┐    ┌─────────────┐              │
│  │  Commit A   │    │  Commit B   │    │ Working Dir │              │
│  │  .ddd files │    │  .ddd files │    │  .ddd files │              │
│  └──────┬──────┘    └──────┬──────┘    └──────┬──────┘              │
│         │                  │                  │                      │
│         └──────────────────┼──────────────────┘                      │
│                            ▼                                         │
│                 ┌──────────────────────┐                            │
│                 │  muscovite-version   │                            │
│                 │  (C++ CLI / libgit2) │                            │
│                 └──────────┬───────────┘                            │
└────────────────────────────┼────────────────────────────────────────┘
                             │ S-expression AND JSON output
                             ▼
┌────────────────────────────────────────────────────────────────────┐
│                       Racket Domain                                 │
│                 ┌──────────────────────┐                           │
│                 │  Parser → IR → Diff  │                           │
│                 │   (semantic engine)  │                           │
│                 └──────────────────────┘                           │
└────────────────────────────────────────────────────────────────────┘

Decision Drivers

  • Compare arbitrary git commits for migration inference
  • Include git metadata (commit, author, timestamp, message) for provenance
  • Output format directly consumable by Racket (S-expressions) AND machine-readable (JSON)
  • Support comparing commit-to-commit, commit-to-HEAD, commit-to-working-tree
  • Handle multi-file projects (multiple .ddd files)
  • Cross-platform (Linux, macOS, Windows)
  • Consistent toolchain: Muscovite is a C++ project; avoid Python dependencies
  • Single build system: Use Conan for all dependencies, CMake for build

Considered Options

  • Embed libgit2 in Racket via FFI: Rejected - complex, no existing bindings
  • Shell out to git from Racket: Rejected - parsing git output is fragile
  • Python CLI with GitPython: Rejected - introduces Python dependency into C++ project
  • C++ CLI with libgit2: Accepted - consistent with project toolchain, Conan package available
  • Go/Rust binary: Rejected - different toolchain than main project

Decision Outcome

Chosen option: "C++ CLI with libgit2", creating muscovite-version tool that:

  1. Takes two version specifiers (commits, branches, tags, HEAD, WORKING)
  2. Extracts all .ddd files from each version
  3. Outputs BOTH S-expression AND JSON formats (user selects via flag)
  4. Includes git metadata for provenance
  5. Built as part of muscovite via CMake, libgit2 via Conan

Positive Consequences

  • Clean separation: Git knowledge in C++, semantic knowledge in Racket
  • Consistent toolchain: No Python dependency, all C++/Conan
  • Single binary distribution
  • Testable with Catch2 (same as rest of muscovite)
  • Metadata enables full provenance tracking
  • Both output formats: S-expr for Racket, JSON for CI/CD and external tools

Negative Consequences

  • More code than Python equivalent
  • Must handle libgit2 memory management
  • Extra process spawn for version extraction

#pragma once

#include <optional>
#include <string>

namespace muscovite::version {

/// Metadata about a model version.
struct VersionInfo {
    std::string ref;                                    // Git reference (commit hash or "WORKING")
    std::optional<std::string> commit_hash;             // Full commit hash, nullopt for working tree
    std::optional<std::string> timestamp;               // ISO timestamp
    std::optional<std::string> author;                  // Author name
    std::optional<std::string> message;                 // Commit message (first line)
    std::unordered_map<std::string, std::string> files; // path -> content mapping
};

/// A pair of versions for diff comparison.
struct VersionPair {
    VersionInfo old_version;
    VersionInfo new_version;
};

} // namespace muscovite::version

class GitLibrary {

    ~GitLibrary() { git_libgit2_shutdown(); }
    GitLibrary(const GitLibrary&) = delete;
    GitLibrary& operator=(const GitLibrary&) = delete;
};

/// Extract model versions from git repository.
class GitExtractor {
public:
    explicit GitExtractor(const std::filesystem::path& repo_path);
    ~GitExtractor();
    
    GitExtractor(const GitExtractor&) = delete;
    GitExtractor& operator=(const GitExtractor&) = delete;

    /// Extract version info for a git reference.
    /// @param ref Git reference: commit hash, branch, tag, HEAD, or "WORKING"
    /// @param ddd_files List of .ddd file paths relative to repo root
    /// @return VersionInfo with files extracted from that reference
    VersionInfo extract_version(const std::string& ref, 
                                const std::vector<std::filesystem::path>& ddd_files);

    /// Find all .ddd files referenced by project.ddd
    std::vector<std::filesystem::path> find_ddd_files(
        const std::filesystem::path& project_path);

    /// Find repository root from current directory
    static std::filesystem::path find_repo_root();

private:
    git_repository* repo_ = nullptr;
    std::filesystem::path repo_root_;

    std::string get_file_at_ref(const std::string& ref, 
                                const std::filesystem::path& file_path);
    std::string get_working_file(const std::filesystem::path& file_path);
};

} // namespace muscovite::version

#include "muscovite/version/git_extractor.hpp"
#include <fstream>

namespace muscovite::version {

GitExtractor::GitExtractor(const std::filesystem::path& repo_path) 
    : repo_root_(repo_path) {
    int error = git_repository_open(&repo_, repo_path.c_str());
    if (error < 0) {
        const git_error* e = git_error_last();
        throw std::runtime_error(
            "Failed to open git repository: " + 
            std::string(e ? e->message : "unknown error"));
    }
}

GitExtractor::~GitExtractor() {
    if (repo_) {
        git_repository_free(repo_);
    }
}

std::filesystem::path GitExtractor::find_repo_root() {
    git_buf buf = GIT_BUF_INIT;
    int error = git_repository_discover(&buf, ".", 0, nullptr);
    if (error < 0) {
        git_buf_dispose(&buf);
        throw std::runtime_error("Not in a git repository");
    }
    
    std::filesystem::path result(buf.ptr);
    git_buf_dispose(&buf);
    
    // discover returns .git path, get parent for worktree
    if (result.filename() == ".git") {
        result = result.parent_path();
    }
    return result;
}

VersionInfo GitExtractor::extract_version(
    const std::string& ref,
    const std::vector<std::filesystem::path>& ddd_files) {
    
    VersionInfo info;
    info.ref = ref;
    
    if (ref == "WORKING") {
        // Working tree (uncommitted changes)
        auto now = std::chrono::system_clock::now();
        auto time_t = std::chrono::system_clock::to_time_t(now);
        char buf[32];
        std::strftime(buf, sizeof(buf), "%Y-%m-%dT%H:%M:%S", std::gmtime(&time_t));
        info.timestamp = std::string(buf) + "+00:00";
        info.message = "(working tree)";
        
        for (const auto& ddd_file : ddd_files) {
            auto full_path = repo_root_ / ddd_file;
            if (std::filesystem::exists(full_path)) {
                info.files[ddd_file.string()] = get_working_file(full_path);
            }
        }
    } else {
        // Git reference (commit, branch, tag)
        git_object* obj = nullptr;
        int error = git_revparse_single(&obj, repo_, ref.c_str());
        if (error < 0) {
            throw std::runtime_error("Invalid git reference: " + ref);
        }
        
        git_commit* commit = nullptr;
        error = git_commit_lookup(&commit, repo_, git_object_id(obj));
        git_object_free(obj);
        
        if (error < 0) {
            throw std::runtime_error("Failed to lookup commit: " + ref);
        }
        
        // Extract metadata
        char oid_str[GIT_OID_HEXSZ + 1];
        git_oid_tostr(oid_str, sizeof(oid_str), git_commit_id(commit));
        info.commit_hash = std::string(oid_str);
        
        const git_signature* author = git_commit_author(commit);
        info.author = std::string(author->name);
        
        auto commit_time = git_commit_time(commit);
        char time_buf[32];
        std::strftime(time_buf, sizeof(time_buf), "%Y-%m-%dT%H:%M:%S", 
                      std::gmtime(&commit_time));
        info.timestamp = std::string(time_buf) + "+00:00";
        
        std::string msg = git_commit_message(commit);
        auto newline = msg.find('\n');
        info.message = (newline != std::string::npos) 
            ? msg.substr(0, newline) 
            : msg;
        
        // Extract files
        for (const auto& ddd_file : ddd_files) {
            try {
                info.files[ddd_file.string()] = get_file_at_ref(ref, ddd_file);
            } catch (...) {
                // File doesn't exist in this version, skip
            }
        }
        
        git_commit_free(commit);
    }
    
    return info;
}

std::string GitExtractor::get_file_at_ref(
    const std::string& ref,
    const std::filesystem::path& file_path) {
    
    git_object* obj = nullptr;
    std::string spec = ref + ":" + file_path.string();
    int error = git_revparse_single(&obj, repo_, spec.c_str());
    
    if (error < 0) {
        throw std::runtime_error("File not found at ref: " + spec);
    }
    
    if (git_object_type(obj) != GIT_OBJECT_BLOB) {
        git_object_free(obj);
        throw std::runtime_error("Not a blob: " + spec);
    }
    
    git_blob* blob = reinterpret_cast<git_blob*>(obj);
    const char* content = static_cast<const char*>(git_blob_rawcontent(blob));
    auto size = git_blob_rawsize(blob);
    
    std::string result(content, size);
    git_object_free(obj);
    return result;
}

std::string GitExtractor::get_working_file(const std::filesystem::path& file_path) {
    std::ifstream file(file_path);
    if (!file) {
        throw std::runtime_error("Cannot open file: " + file_path.string());
    }
    std::stringstream buffer;
    buffer << file.rdbuf();
    return buffer.str();
}

std::vector<std::filesystem::path> GitExtractor::find_ddd_files(
    const std::filesystem::path& project_path) {
    
    std::vector<std::filesystem::path> ddd_files;
    auto project_dir = project_path.parent_path();
    
    // Read project file for INCLUDE directives
    if (std::filesystem::exists(project_path)) {
        std::ifstream file(project_path);
        std::string content((std::istreambuf_iterator<char>(file)),
                            std::istreambuf_iterator<char>());
        
        std::regex include_regex(R"(INCLUDE\s+"([^"]+)")");
        std::smatch match;
        std::string::const_iterator search_start(content.cbegin());
        
        while (std::regex_search(search_start, content.cend(), match, include_regex)) {
            auto inc_path = project_dir / match[1].str();
            if (std::filesystem::exists(inc_path) && inc_path.extension() == ".ddd") {
                ddd_files.push_back(std::filesystem::relative(inc_path, repo_root_));
            }
            search_start = match.suffix().first;
        }
    }
    
    // Also find all .ddd files in project directory
    for (const auto& entry : std::filesystem::recursive_directory_iterator(project_dir)) {
        if (entry.path().extension() == ".ddd") {
            auto rel_path = std::filesystem::relative(entry.path(), repo_root_);
            if (std::find(ddd_files.begin(), ddd_files.end(), rel_path) == ddd_files.end()) {
                ddd_files.push_back(rel_path);
            }
        }
    }
    
    std::sort(ddd_files.begin(), ddd_files.end());
    return ddd_files;
}

} // namespace muscovite::version

#include "muscovite/version/output_formatter.hpp"

namespace muscovite::version {

std::string format_version_pair(const VersionPair& pair, OutputFormat format) {
    switch (format) {
        case OutputFormat::SExpr: return format_sexp(pair);
        case OutputFormat::Json:  return format_json(pair);
    }
    return "";
}

namespace {

std::string escape_string(const std::string& s) {
    std::ostringstream result;
    for (char c : s) {
        switch (c) {
            case '\\': result << "\\\\"; break;
            case '"':  result << "\\\""; break;
            case '\n': result << "\\n"; break;
            case '\r': result << "\\r"; break;
            case '\t': result << "\\t"; break;
            default:   result << c; break;
        }
    }
    return result.str();
}

std::string format_version_sexp(const VersionInfo& v, const std::string& label) {
    std::ostringstream ss;
    ss << "  (" << label << "\n";
    ss << "    (ref \"" << v.ref << "\")\n";
    
    if (v.commit_hash) {
        ss << "    (commit-hash \"" << *v.commit_hash << "\")\n";
    } else {
        ss << "    (commit-hash #f)\n";
    }
    
    if (v.timestamp) {
        ss << "    (timestamp \"" << *v.timestamp << "\")\n";
    } else {
        ss << "    (timestamp #f)\n";
    }
    
    if (v.author) {
        ss << "    (author \"" << escape_string(*v.author) << "\")\n";
    } else {
        ss << "    (author #f)\n";
    }
    
    if (v.message) {
        ss << "    (message \"" << escape_string(*v.message) << "\")\n";
    } else {
        ss << "    (message #f)\n";
    }
    
    ss << "    (files\n";
    for (const auto& [path, content] : v.files) {
        ss << "      (\"" << path << "\" . \"" << escape_string(content) << "\")\n";
    }
    ss << "    ))";
    
    return ss.str();
}

} // anonymous namespace

std::string format_sexp(const VersionPair& pair) {
    std::ostringstream ss;
    ss << "(version-pair\n";
    ss << format_version_sexp(pair.old_version, "old") << "\n";
    ss << format_version_sexp(pair.new_version, "new") << ")\n";
    return ss.str();
}

std::string format_json(const VersionPair& pair) {
    auto format_version = [](const VersionInfo& v) -> nlohmann::json {
        nlohmann::json files = nlohmann::json::object();
        for (const auto& [path, content] : v.files) {
            files[path] = content;
        }
        
        return {
            {"ref", v.ref},
            {"commit_hash", v.commit_hash.value_or(nlohmann::json())},
            {"timestamp", v.timestamp.value_or(nlohmann::json())},
            {"author", v.author.value_or(nlohmann::json())},
            {"message", v.message.value_or(nlohmann::json())},
            {"files", files}
        };
    };
    
    nlohmann::json result = {
        {"old", format_version(pair.old_version)},
        {"new", format_version(pair.new_version)}
    };
    
    return result.dump(2);
}

} // namespace muscovite::version

#lang racket/base

(require racket/port
         racket/system
         racket/string
         json
         "../parser/parser.rkt")

(provide read-version-pair
         version-pair-old-ir
         version-pair-new-ir
         get-versions-from-git)

;; Struct for parsed version with IR
(struct parsed-version
  (ref commit-hash timestamp author message ir)
  #:transparent)

(struct version-pair
  (old new)
  #:transparent)

;; Read S-expression version pair from port
(define (read-version-pair-sexp port)
  (define sexp (read port))
  (unless (and (list? sexp) (eq? (car sexp) 'version-pair))
    (error 'read-version-pair "Invalid version-pair format"))
  
  (define old-sexp (assoc 'old (cdr sexp)))
  (define new-sexp (assoc 'new (cdr sexp)))
  
  (version-pair
   (parse-version-sexp (cdr old-sexp))
   (parse-version-sexp (cdr new-sexp))))

;; Parse single version from S-expression
(define (parse-version-sexp sexp)
  (define (get-field name)
    (define pair (assoc name sexp))
    (and pair (cadr pair)))
  
  (define files (cdr (assoc 'files sexp)))
  
  ;; Parse all .ddd files into combined IR
  (define ir (parse-version-files files))
  
  (parsed-version
   (get-field 'ref)
   (get-field 'commit-hash)
   (get-field 'timestamp)
   (get-field 'author)
   (get-field 'message)
   ir))

;; Parse files into IR
(define (parse-version-files file-alist)
  ;; Each file is (path . content)
  ;; Parse each .ddd file and merge contexts
  (define all-contexts
    (for/fold ([contexts '()])
              ([file-pair (in-list file-alist)])
      (define path (car file-pair))
      (define content (cdr file-pair))
      (with-handlers ([exn:fail? 
                       (λ (e) 
                         (eprintf "Warning: Failed to parse ~a: ~a\n" 
                                  path (exn-message e))
                         contexts)])
        (define model (parse-string content))
        (append contexts (ddd-model-contexts model)))))
  
  ;; Return merged model
  (ddd-model all-contexts))

;; Call CLI tool and read result

    (subprocess #f #f #f tool-path old-ref new-ref 
                "--project" project format-arg))
  
  (define output (port->string stdout))
  (define errors (port->string stderr))
  (subprocess-wait proc)
  
  (define exit-code (subprocess-status proc))
  (unless (zero? exit-code)
    (error 'get-versions-from-git "Tool failed: ~a" errors))
  
  (close-input-port stdout)
  (close-input-port stderr)
  (close-output-port stdin)
  
  (if (eq? format 'json)
      (read-version-pair-json (open-input-string output))
      (read-version-pair-sexp (open-input-string output))))

;; Convenience: get IRs for diff
(define (version-pair-old-ir pair)
  (parsed-version-ir (version-pair-old pair)))

(define (version-pair-new-ir pair)
  (parsed-version-ir (version-pair-new pair)))

(define (migrate-command args)
  (define old-ref (hash-ref args 'old-ref))
  (define new-ref (hash-ref args 'new-ref "WORKING"))
  (define project (hash-ref args 'project "project.ddd"))
  
  ;; Get versions from git
  (printf "Extracting versions: ~a → ~a\n" old-ref new-ref)
  (define pair (get-versions-from-git old-ref new-ref #:project project))
  
  ;; Extract IRs
  (define old-ir (version-pair-old-ir pair))
  (define new-ir (version-pair-new-ir pair))
  
  (printf "Old version: ~a (~a contexts)\n" 
          (parsed-version-ref (version-pair-old pair))
          (length (ddd-model-contexts old-ir)))
  (printf "New version: ~a (~a contexts)\n"
          (parsed-version-ref (version-pair-new pair))
          (length (ddd-model-contexts new-ir)))
  
  ;; Run semantic diff
  (printf "Computing semantic diff...\n")
  (define operations (diff-model old-ir new-ir))
  
  (printf "Found ~a operations:\n" (length operations))
  (for ([op (in-list operations)])
    (printf "  [~a] ~a\n" 
            (migration-op-safety op)
            (migration-op-description op)))
  
  ;; Create migration plan
  (define plan (diff->migration-plan operations old-ir new-ir))
  
  ;; Write migration plan
  (define plan-path (format "migrations/~a.migration" (migration-plan-name plan)))
  (write-migration-plan plan plan-path)
  (printf "Migration plan written to: ~a\n" plan-path))

When This Decision Applies

  • When comparing model versions across git history
  • When generating migrations for deployment
  • When reviewing changes before release
  • When building the muscovite-version tool

When This Decision Does NOT Apply

  • Comparing deployed database schema (use ADR 0007 verification)
  • Diffing generated code (use git diff directly)
  • Real-time change detection (this is batch-oriented)

Enforcement

  • muscovite-version MUST output valid S-expressions AND JSON (both formats supported)
  • Git metadata MUST be included for provenance
  • All .ddd files in project MUST be included
  • Parse errors in individual files MUST warn but not fail entirely
  • Tool MUST be built as part of standard cmake --build --preset debug

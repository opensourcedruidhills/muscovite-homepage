Migration SQL Generation via Sqitch

Migration SQL Generation via Sqitch

  • Status: accepted
  • Deciders: Project maintainers
  • Date: 2026-01-09
  • Tags: sqitch, sql, migration, postgresql, schema-evolution
  • Related: Migration Safety Policies, Cross-Layer Verification

Technical Story: Generate Sqitch-compatible deploy/revert/verify scripts from approved migration plans.

Context and Problem Statement

Migration IR must be rendered into deployable SQL scripts. The existing Sqitch generator (sqitch-unified.rkt) performs full regeneration—it always produces the complete schema from scratch. This works for greenfield but not for schema evolution:

  1. Full regeneration loses data: Existing tables would be dropped
  2. No incremental changes: Cannot add a single column without regenerating everything
  3. No rollback awareness: Revert scripts are generic DROP statements
  4. No verification awareness: Verify scripts don't check for specific changes

The new generator must:

  • Accept a migration plan (from ADR 0004) as input
  • Generate incremental deploy/revert/verify scripts
  • Produce deterministic change names for Sqitch
  • Support safe rollbacks with proper revert scripts

Decision Drivers

  • Deterministic and reversible SQL generation
  • Preserve all semantic operations from migration IR
  • Compatible with reviewable Migration IR from ADR 0004
  • Only execute on approved migrations (from ADR 0005)
  • Versioned and deterministic change names
  • Integrate with existing Sqitch infrastructure

Considered Options

  • Replace sqitch-unified.rkt entirely: Rejected - still need full generation for new projects
  • External migration framework (Flyway, Liquibase): Rejected - already invested in Sqitch
  • Add incremental mode to existing generator: Rejected - too much coupling
  • New generator that consumes migration-plan: Accepted - clean separation

Decision Outcome

Chosen option: "New generator that consumes migration-plan", creating sqitch-from-migration.rkt that:

  1. Takes an approved migration plan as input
  2. Generates a single Sqitch change with deploy/revert/verify
  3. Produces deterministic change name based on plan hash
  4. Supports both single-operation and batched changes

Positive Consequences

  • Safe, reversible migrations
  • Compatible with CI/CD pipelines
  • Fully deterministic output
  • Coexists with existing full-regeneration mode
  • Each migration is a reviewable, atomic unit

Negative Consequences

  • Dependency on Sqitch (acceptable, already committed)
  • SQL must follow PostgreSQL conventions
  • Two code paths (full regen vs incremental)

Implementation

Phase 1: Migration-to-SQL Mapping

Create src/dslgen/generators/postgres/sqitch-from-migration.rkt:

#lang racket/base

(require "../../migration/migration-types.rkt")

(provide generate-sqitch-from-migration)

;; Main entry point
(define (generate-sqitch-from-migration approved-plan output-dir)
  (define change-name (plan->change-name approved-plan))
  (define deploy-sql (generate-deploy-sql approved-plan))
  (define revert-sql (generate-revert-sql approved-plan))
  (define verify-sql (generate-verify-sql approved-plan))
  
  ;; Write Sqitch files
  (write-sqitch-change output-dir change-name deploy-sql revert-sql verify-sql)
  
  ;; Update sqitch.plan
  (append-to-sqitch-plan output-dir change-name (plan-dependencies approved-plan)))

;; Generate deterministic change name
(define (plan->change-name plan)
  (define hash-prefix (substring (migration-plan-target-version plan) 0 8))
  (define name (migration-plan-name plan))
  (format "~a-~a" name hash-prefix))

Phase 2: Operation-to-SQL Generators

;; Dispatch to appropriate SQL generator
(define (operation->deploy-sql op)
  (match op
    [(add-entity-op uuid safety desc requires ctx-name entity-name entity-uuid fields pk parent)
     (generate-create-table ctx-name entity-name fields pk parent)]
    
    [(remove-entity-op uuid safety desc requires ctx-name entity-name cascade?)
     (if cascade?
         (format "DROP TABLE IF EXISTS ~a.~a CASCADE;" ctx-name entity-name)
         (format "DROP TABLE IF EXISTS ~a.~a;" ctx-name entity-name))]
    
    [(rename-entity-op uuid safety desc requires ctx-name old-name new-name)
     (format "ALTER TABLE ~a.~a RENAME TO ~a;" ctx-name old-name new-name)]
    
    [(add-field-op uuid safety desc requires entity-uuid field-name field-uuid 
                   field-type nullable? default-value)
     (define entity-path (uuid->entity-path entity-uuid))
     (define null-clause (if nullable? "" " NOT NULL"))
     (define default-clause (if default-value (format " DEFAULT ~a" default-value) ""))
     (format "ALTER TABLE ~a ADD COLUMN ~a ~a~a~a;"
             entity-path field-name (type->sql field-type) null-clause default-clause)]
    
    [(remove-field-op uuid safety desc requires entity-uuid field-name)
     (define entity-path (uuid->entity-path entity-uuid))
     (format "ALTER TABLE ~a DROP COLUMN ~a;" entity-path field-name)]
    
    [(rename-field-op uuid safety desc requires entity-uuid old-name new-name)
     (define entity-path (uuid->entity-path entity-uuid))
     (format "ALTER TABLE ~a RENAME COLUMN ~a TO ~a;" entity-path old-name new-name)]
    
    [(change-type-op uuid safety desc requires entity-uuid field-name old-type new-type conversion)
     (define entity-path (uuid->entity-path entity-uuid))
     (if conversion
         (format "ALTER TABLE ~a ALTER COLUMN ~a TYPE ~a USING ~a;"
                 entity-path field-name (type->sql new-type) conversion)
         (format "ALTER TABLE ~a ALTER COLUMN ~a TYPE ~a;"
                 entity-path field-name (type->sql new-type)))]
    
    [(change-nullable-op uuid safety desc requires entity-uuid field-name old new default-for-nulls)
     (define entity-path (uuid->entity-path entity-uuid))
     (if new  ;; becoming nullable
         (format "ALTER TABLE ~a ALTER COLUMN ~a DROP NOT NULL;" entity-path field-name)
         (string-append
          (if default-for-nulls
              (format "UPDATE ~a SET ~a = ~a WHERE ~a IS NULL;\n"
                      entity-path field-name default-for-nulls field-name)
              "")
          (format "ALTER TABLE ~a ALTER COLUMN ~a SET NOT NULL;" entity-path field-name)))]
    
    [(add-relationship-op uuid safety desc requires src-uuid tgt-uuid rel-name rel-uuid card on-delete)
     (define src-path (uuid->entity-path src-uuid))
     (define tgt-path (uuid->entity-path tgt-uuid))
     (define fk-col (format "~a_id" rel-name))
     (define on-delete-clause 
       (case on-delete
         [(cascade) " ON DELETE CASCADE"]
         [(set-null) " ON DELETE SET NULL"]
         [(restrict) " ON DELETE RESTRICT"]
         [else ""]))
     (format "ALTER TABLE ~a ADD CONSTRAINT fk_~a FOREIGN KEY (~a) REFERENCES ~a(id)~a;"
             src-path rel-name fk-col tgt-path on-delete-clause)]
    
    [(add-index-op uuid safety desc requires entity-uuid index-name columns unique?)
     (define entity-path (uuid->entity-path entity-uuid))
     (define unique-clause (if unique? "UNIQUE " ""))
     (format "CREATE ~aINDEX ~a ON ~a (~a);"
             unique-clause index-name entity-path (string-join columns ", "))]))

Phase 3: Revert SQL Generation

;; Generate revert (inverse) SQL for each operation
(define (operation->revert-sql op)
  (match op
    [(add-entity-op uuid safety desc requires ctx-name entity-name entity-uuid fields pk parent)
     (format "DROP TABLE IF EXISTS ~a.~a CASCADE;" ctx-name entity-name)]
    
    [(remove-entity-op uuid safety desc requires ctx-name entity-name cascade?)
     ;; Cannot truly revert a DROP - would need to recreate
     ;; This is a placeholder that will fail safely
     (format "-- Cannot revert DROP TABLE ~a.~a - manual intervention required" 
             ctx-name entity-name)]
    
    [(rename-entity-op uuid safety desc requires ctx-name old-name new-name)
     (format "ALTER TABLE ~a.~a RENAME TO ~a;" ctx-name new-name old-name)]
    
    [(add-field-op uuid safety desc requires entity-uuid field-name field-uuid 
                   field-type nullable? default-value)
     (define entity-path (uuid->entity-path entity-uuid))
     (format "ALTER TABLE ~a DROP COLUMN IF EXISTS ~a;" entity-path field-name)]
    
    [(remove-field-op uuid safety desc requires entity-uuid field-name)
     ;; Cannot truly revert a DROP COLUMN - data is lost
     (format "-- Cannot revert DROP COLUMN ~a - manual intervention required" field-name)]
    
    [(rename-field-op uuid safety desc requires entity-uuid old-name new-name)
     (define entity-path (uuid->entity-path entity-uuid))
     (format "ALTER TABLE ~a RENAME COLUMN ~a TO ~a;" entity-path new-name old-name)]
    
    [(change-type-op uuid safety desc requires entity-uuid field-name old-type new-type conversion)
     (define entity-path (uuid->entity-path entity-uuid))
     ;; Attempt reverse conversion - may not always work
     (format "ALTER TABLE ~a ALTER COLUMN ~a TYPE ~a;" 
             entity-path field-name (type->sql old-type))]
    
    [(add-relationship-op uuid safety desc requires src-uuid tgt-uuid rel-name rel-uuid card on-delete)
     (define src-path (uuid->entity-path src-uuid))
     (format "ALTER TABLE ~a DROP CONSTRAINT IF EXISTS fk_~a;" src-path rel-name)]
    
    [(add-index-op uuid safety desc requires entity-uuid index-name columns unique?)
     (format "DROP INDEX IF EXISTS ~a;" index-name)]))

Phase 4: Verify SQL Generation

;; Generate verification SQL (returns true if migration applied correctly)
(define (operation->verify-sql op)
  (match op
    [(add-entity-op uuid safety desc requires ctx-name entity-name _ _ _ _)
     (format "SELECT 1 FROM information_schema.tables WHERE table_schema = '~a' AND table_name = '~a';"
             ctx-name entity-name)]
    
    [(remove-entity-op uuid safety desc requires ctx-name entity-name _)
     (format "SELECT NOT EXISTS (SELECT 1 FROM information_schema.tables WHERE table_schema = '~a' AND table_name = '~a');"
             ctx-name entity-name)]
    
    [(rename-entity-op uuid safety desc requires ctx-name old-name new-name)
     (format "SELECT 1 FROM information_schema.tables WHERE table_schema = '~a' AND table_name = '~a';"
             ctx-name new-name)]
    
    [(add-field-op uuid safety desc requires entity-uuid field-name _ _ _ _)
     (define-values (schema table) (uuid->schema-table entity-uuid))
     (format "SELECT 1 FROM information_schema.columns WHERE table_schema = '~a' AND table_name = '~a' AND column_name = '~a';"
             schema table field-name)]
    
    [(remove-field-op uuid safety desc requires entity-uuid field-name)
     (define-values (schema table) (uuid->schema-table entity-uuid))
     (format "SELECT NOT EXISTS (SELECT 1 FROM information_schema.columns WHERE table_schema = '~a' AND table_name = '~a' AND column_name = '~a');"
             schema table field-name)]
    
    [(add-relationship-op uuid safety desc requires src-uuid _ rel-name _ _ _)
     (define-values (schema table) (uuid->schema-table src-uuid))
     (format "SELECT 1 FROM information_schema.table_constraints WHERE table_schema = '~a' AND table_name = '~a' AND constraint_name = 'fk_~a';"
             schema table rel-name)]))

;; Combine all verify statements
(define (generate-verify-sql plan)
  (define stmts 
    (for/list ([op (migration-plan-operations plan)])
      (operation->verify-sql op)))
  (string-join stmts "\n"))

Phase 5: Sqitch Integration

;; Write Sqitch change files
(define (write-sqitch-change output-dir change-name deploy revert verify)
  (define deploy-path (build-path output-dir "deploy" (format "~a.sql" change-name)))
  (define revert-path (build-path output-dir "revert" (format "~a.sql" change-name)))
  (define verify-path (build-path output-dir "verify" (format "~a.sql" change-name)))
  
  (make-directory* (build-path output-dir "deploy"))
  (make-directory* (build-path output-dir "revert"))
  (make-directory* (build-path output-dir "verify"))
  
  (call-with-output-file deploy-path
    (λ (out) (display deploy out))
    #:exists 'replace)
  
  (call-with-output-file revert-path
    (λ (out) (display revert out))
    #:exists 'replace)
  
  (call-with-output-file verify-path
    (λ (out) (display verify out))
    #:exists 'replace))

;; Append to sqitch.plan
(define (append-to-sqitch-plan output-dir change-name dependencies)
  (define plan-path (build-path output-dir "sqitch.plan"))
  (define dep-str 
    (if (null? dependencies)
        ""
        (format " [~a]" (string-join dependencies " "))))
  (define entry (format "~a~a ~a" change-name dep-str (current-rfc3339)))
  
  (call-with-output-file plan-path
    (λ (out) (displayln entry out))
    #:exists 'append))

AI Agent Guidance

When This Decision Applies

  • When generating SQL from approved migration plans
  • When implementing incremental schema changes
  • When maintaining Sqitch change history

When This Decision Does NOT Apply

  • Full schema generation for new projects (use sqitch-unified.rkt)
  • Runtime SQL execution
  • Non-PostgreSQL databases

Enforcement

  • MUST only accept approved migration plans (from ADR 0005)
  • MUST generate all three files (deploy/revert/verify)
  • MUST produce deterministic change names
  • MUST update sqitch.plan with dependencies
  • Destructive operations MUST include comment warning in revert SQL

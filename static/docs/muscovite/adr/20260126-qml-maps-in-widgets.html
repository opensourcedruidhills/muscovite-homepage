QML Maps in Qt Widgets Application

QML Maps in Qt Widgets Application

  • Status: accepted
  • Deciders: jolo
  • Date: 2026-01-26

Context and Problem Statement

Muscovite generates Qt Widgets desktop applications (per ADR: Qt Widgets Only - No QML). However, Qt's best map visualization component (QtLocation::Map) is QML-only. We need a strategy for displaying interactive maps within our Widgets-based architecture.

Decision Drivers

  • Qt Widgets is mandatory (licensing, architectural consistency)
  • Map visualization is essential for spatial data
  • Qt Location Map is QML-only, no pure C++ widget equivalent
  • Users need pan, zoom, and feature overlay capabilities

Considered Options

  1. QQuickWidget embedding - Embed QML map in a QWidget container
  2. Third-party map library - Use a C++ map widget (MapTiler, Marble, etc.)
  3. Web-based map - Embed Leaflet/OpenLayers via QWebEngineView
  4. No map widget - Only text coordinate display

Decision Outcome

Chosen option: QQuickWidget embedding

Use QQuickWidget to embed a QML Map component within the otherwise pure-Widgets application. This is the Qt-sanctioned approach for mixing QML and Widgets.

Architecture

┌─────────────────────────────────────────┐
│  EntityDetailWidget (QWidget)           │
│  ├── QTabWidget                         │
│  │   ├── Tab 1: Form fields (QWidget)   │
│  │   └── Tab 2: Map view                │
│  │       └── QQuickWidget               │
│  │           └── QML Map component      │
│  │               └── Qt Location        │
└─────────────────────────────────────────┘

QML Component (Embedded Only)

The ONLY QML in the system is for map rendering. No business logic in QML.

// resources/qml/MapView.qml
import QtQuick
import QtLocation
import QtPositioning

Item {
    id: root
    
    property alias center: map.center
    property alias zoomLevel: map.zoomLevel
    
    // Signal to C++ when marker clicked
    signal featureClicked(string id)
    
    Map {
        id: map
        anchors.fill: parent
        plugin: Plugin { name: "osm" }  // OpenStreetMap
        
        // Overlay items populated from C++
        MapItemView {
            model: featureModel
            delegate: MapQuickItem {
                coordinate: QtPositioning.coordinate(model.latitude, model.longitude)
                sourceItem: Rectangle {
                    width: 12; height: 12
                    color: model.selected ? "red" : "blue"
                    radius: 6
                }
                MouseArea {
                    anchors.fill: parent
                    onClicked: root.featureClicked(model.id)
                }
            }
        }
    }
}

C++ Integration

class SpatialEntityWidget : public QWidget {
    Q_OBJECT
public:
    explicit SpatialEntityWidget(QWidget* parent = nullptr);
    
    void setFeatures(const QList<SpatialFeature>& features);
    void setCenter(double lat, double lon);
    void setZoomLevel(int level);
    
signals:
    void featureSelected(const QString& id);
    
private:
    QQuickWidget* mapWidget_{};
    QStandardItemModel* featureModel_{};
};

Consequences

Good:

  • Official Qt approach for this use case
  • Full Qt Location capabilities (tiles, plugins, gestures)
  • No external dependencies beyond Qt
  • Works offline with cached tiles

Bad:

  • Requires Qt6::Quick and Qt6::QuickWidgets dependencies
  • QML file must be distributed with app
  • Slightly more complex build (QML resources)

CMake Integration

# Generated CMakeLists.txt for projects with spatial capability
find_package(Qt6 REQUIRED COMPONENTS
    Widgets
    Quick
    QuickWidgets
    Location
    Positioning
)

qt_add_executable(${PROJECT}_client
    src/main.cpp
    # ... other sources
)

qt_add_qml_module(${PROJECT}_client
    URI MapComponents
    VERSION 1.0
    QML_FILES
        resources/qml/MapView.qml
)

target_link_libraries(${PROJECT}_client PRIVATE
    Qt6::Widgets
    Qt6::Quick
    Qt6::QuickWidgets
    Qt6::Location
    Qt6::Positioning
)

AI Agent Guidance

When this applies:

  • Generating Qt client code for entities with spatial fields
  • Setting up CMakeLists.txt for projects with CAPABILITIES { spatial }
  • Creating map visualization widgets

Critical rules:

  1. QML is ONLY for MapView.qml - no business logic
  2. All data flow is C++ → QML model → display
  3. QQuickWidget is embedded in QWidget hierarchy
  4. featureModel is a QStandardItemModel owned by C++

Files affected:

  • src/dslgen/generators/qt/widgets-views.rkt - Generate SpatialEntityWidget
  • src/dslgen/generators/cmake/qt-cmake.rkt - Add Qt6::Quick dependencies
  • src/muscframework/qt/resources/qml/MapView.qml - Framework QML file

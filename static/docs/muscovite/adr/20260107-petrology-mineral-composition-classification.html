Petrology Mineral Composition and Rock Classification

Petrology Mineral Composition and Rock Classification

  • Status: accepted
  • Deciders: Domain modeling team
  • Date: 2026-01-07
  • Tags: petrology, classification, QAPF, TAS, IUGS

Context and Problem Statement

Igneous rocks are classified based on their mineral composition. The percentage of key minerals determines the rock name. We need to model the relationship between rocks and their constituent minerals, and support standard classification schemes (QAPF, TAS).

Decision Drivers

  • IUGS recommends QAPF diagram for coarse-grained igneous rocks
  • TAS classification used when modal analysis not possible (fine-grained/volcanic)
  • Mineral percentages determine rock classification
  • Support for essential vs accessory mineral designation
  • Sedimentary rocks also have mineral compositions (detrital, authigenic)

Considered Options

  • Embed mineral percentages directly in rock entities (current approach via MineralMode)
  • Junction table linking Rock to Mineral with percentage and role
  • Hybrid: Keep MineralMode for summary, add junction for detailed composition

Decision Outcome

Chosen option: "Hybrid approach", because:

  1. MineralMode value object provides quick access to common minerals
  2. Junction table enables detailed composition with FK to Mineralogy.Mineral
  3. Supports QAPF/TAS classification based on normalized percentages
  4. Enables queries like "find rocks containing >10% olivine"

Rock Mineral Composition Junction Entity

ENTITY RockMineralComposition IDENTIFIED_BY id (
    id              UUID        NOT NULL,
    rock_type       VARCHAR(30) NOT NULL,
    rock_id         UUID        NOT NULL,
    mineral_id      UUID        NOT NULL,
    percentage      DECIMAL     NOT NULL,
    role            VARCHAR(30) NOT NULL,
    determination   VARCHAR(30),
    notes           TEXT
)
CHECK percentage_valid: percentage >= 0 AND percentage <= 100
CHECK role_valid: role IN ('essential', 'accessory', 'secondary', 'xenocrystic')
CHECK rock_type_valid: rock_type IN ('igneous', 'sedimentary', 'metamorphic')
RELATIONSHIPS {
    mineral -> Mineral REQUIRED ONE;
}

QAPF Classification Lookup

The QAPF diagram uses normalized percentages of:

  • Q = Quartz
  • A = Alkali feldspar (orthoclase, sanidine, microcline, perthite, anorthoclase, albite An<5)
  • P = Plagioclase (An≥5)
  • F = Feldspathoid (nepheline, leucite, sodalite, etc.)

Note: Q and F are mutually exclusive (silica oversaturated vs undersaturated).

ENTITY QAPFClassification IDENTIFIED_BY id (
    id              UUID        NOT NULL,
    field_number    VARCHAR(10) NOT NULL @unique,
    rock_name       VARCHAR(80) NOT NULL,
    q_min           DECIMAL,
    q_max           DECIMAL,
    a_min           DECIMAL,
    a_max           DECIMAL,
    p_min           DECIMAL,
    p_max           DECIMAL,
    f_min           DECIMAL,
    f_max           DECIMAL,
    emplacement     VARCHAR(30) NOT NULL,
    description     TEXT
)
CHECK emplacement_valid: emplacement IN ('plutonic', 'volcanic')

Key QAPF fields (plutonic rocks): | Field | Rock Name | Q | A | P | Notes | |-------|-----------|---|---|---|-------| | 1a | Quartzolite | 90-100 | - | - | Almost pure quartz | | 2 | Alkali feldspar granite | 20-60 | 90-100 | 0-10 | | | 3a | Syenogranite | 20-60 | 65-90 | 10-35 | | | 3b | Monzogranite | 20-60 | 35-65 | 35-65 | | | 4 | Granodiorite | 20-60 | 10-35 | 65-90 | | | 5 | Tonalite/Trondhjemite | 20-60 | 0-10 | 90-100 | | | 6* | Alkali feldspar syenite | 0-5 | 90-100 | 0-10 | Q<5%, no F | | 7* | Syenite | 0-5 | 65-90 | 10-35 | | | 8* | Monzonite | 0-5 | 35-65 | 35-65 | | | 9* | Monzodiorite | 0-5 | 10-35 | 65-90 | | | 10* | Diorite/Gabbro | 0-5 | 0-10 | 90-100 | Diorite if Plag An<50 |

TAS Classification

Total Alkali-Silica diagram for volcanic rocks:

  • X-axis: SiO₂ (wt%)
  • Y-axis: Na₂O + K₂O (wt%)
ENTITY TASClassification IDENTIFIED_BY id (
    id              UUID        NOT NULL,
    field_name      VARCHAR(50) NOT NULL @unique,
    rock_name       VARCHAR(80) NOT NULL,
    sio2_min        DECIMAL     NOT NULL,
    sio2_max        DECIMAL     NOT NULL,
    alkali_min      DECIMAL     NOT NULL,
    alkali_max      DECIMAL     NOT NULL,
    silica_category VARCHAR(20),
    description     TEXT
)
CHECK silica_valid: silica_category IN ('ultrabasic', 'basic', 'intermediate', 'acid') OR silica_category IS NULL

Key TAS fields: | Field | Rock Name | SiO₂ | Na₂O+K₂O | Category | |-------|-----------|------|----------|----------| | Basalt | Basalt | 45-52 | 0-5 | Basic | | Andesite | Andesite | 52-63 | 2-6 | Intermediate | | Dacite | Dacite | 63-69 | 3-7 | Acid | | Rhyolite | Rhyolite | 69-77 | 5-10 | Acid | | Phonolite | Phonolite | 52-63 | 9-14 | Intermediate/alkaline | | Trachyte | Trachyte | 60-69 | 7-12 | Intermediate/alkaline |

Positive Consequences

  • Rocks can be classified by mineral composition using standard IUGS schemes
  • Detailed mineral percentages enable accurate QAPF/TAS placement
  • FK to Mineralogy.Mineral ensures data consistency
  • Supports queries across rock-mineral relationships

Negative Consequences

  • Additional tables increase complexity
  • QAPF field boundaries require careful polygon definition
  • Some rock names are ambiguous in classification schemes

Pure Function Compilation — DECIDE Block to C++ DomainService Methods

Pure Function Compilation — DECIDE Block to C++ DomainService Methods

  • Status: accepted
  • Date: 2026-03-07
  • Deciders: Johannes Lochmann
  • Tags: decide, domain-service, compilation, behavioral-sovereignty

Context

The DECIDE block (ADR 20260307-command-behavioral-evolution) contains pure functional logic that must be compiled to C++. The original proposal suggested a separate "pure C++ class" per command. After assessment, the DomainService (ADR 20260307-domain-service-layer) is the correct generation target — it already serves as the orchestration layer between gRPC handlers and persistence.

The compilation must guarantee purity: the generated DECIDE method cannot access databases, network, or mutable global state. Only the entity's current state and the command's input parameters are available.

Decision

Generate DECIDE blocks as const methods on the DomainService class:

// Generated into DomainService
struct DischargeDecision {
    std::vector<DomainEvent> events;
    std::vector<EffectIntent> effects;
};

[[nodiscard]] DischargeDecision decide_discharge_patient(
    const Patient& entity,
    const DischargePatientParams& params) const;

Compilation rules for expression AST → C++:

AST Node C++ Output
let-stmt const auto name = expr;
if-expr (cond) ? then_expr : else_expr (or if/else for complex)
binary-expr(+,-,*,/) Arithmetic operators
binary-expr(and,or) &&, ||
map-expr std::ranges::transform + std::ranges::to<std::vector>()
filter-expr std::ranges::filter + std::ranges::to<std::vector>()
reduce-expr std::ranges::fold_left
call-expr(SUM) std::ranges::fold_left(range, 0, std::plus{})
emit-stmt Push DomainEvent to result vector
plan-stmt Push EffectIntent to result vector

DomainService orchestration (generated):

void PatientDomainService::discharge_patient(
    UnitOfWork& uow,
    const boost::uuids::uuid& patient_id,
    const DischargePatientParams& params) {
    
    auto patient = uow.repository<Patient>().find_by_id(patient_id);
    
    // REQUIRES checks
    if (!(patient.status == "admitted")) {
        throw PreconditionFailed("active_admission", "patient.status must be 'admitted'");
    }
    
    // DECIDE — pure computation
    auto decision = decide_discharge_patient(patient, params);
    
    // ENSURES checks
    if (!(patient_after.status == "discharged")) {
        throw PostconditionFailed("discharged_status");
        // UnitOfWork destructor rolls back
    }
    
    // Apply intents
    for (const auto& event : decision.events) {
        uow.add_event(event);
    }
    for (const auto& effect : decision.effects) {
        uow.plan_effect(effect);
    }
}

Purity enforcement: The decide_* method is const, takes only value parameters, returns a value type. No reference to UnitOfWork, no repository access, no I/O handles. The compiler enforces this via const qualification.

Consequences

  • No new "compiled" generator target — DECIDE compiles into DomainService methods
  • DomainService generator (FTR-270) must support both stub and DECIDE-based commands
  • expression->cpp emitter (currently string-based in ACL adapter) must be rewritten to walk AST nodes
  • New framework types needed: DecisionResult, EffectIntent, PreconditionFailed, PostconditionFailed
  • C++23 ranges used extensively — requires -std=c++23 (already enforced)
  • Generated code is fully deterministic and testable without any infrastructure

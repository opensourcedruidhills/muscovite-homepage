Release: Skip Unchanged Layers by Default

Release: Skip Unchanged Layers by Default

  • Status: accepted
  • Deciders: Johannes Lochmann
  • Date: 2026-03-16
  • Tags: release, ci, quality-gates

Technical Story: v1.67.0 release attempt 3 failed on clang-tidy due to a pre-existing boost <chrono> header issue, despite zero C++ changes in the iteration. FTR-443 --skip-unchanged-layers saved the release but had to be manually specified.

Context and Problem Statement

release.sh runs all quality gates unconditionally by default. When an iteration contains no C++ changes, the C++ build, ctest, and clang-tidy gates still run. This creates two problems:

  1. Environment fragility: Pre-existing toolchain issues (missing headers, compiler version mismatches) can block releases that have nothing to do with C++.
  2. Wasted compute: Full C++ build + clang-tidy takes ~25 min on the CI machine. For DSL-only iterations, this is pure overhead.

In v1.67.0 (7 FTRs, all in grammar/IR/parser/scripts), the clang-tidy gate failed due to a missing boost <chrono> header — a pre-existing environment issue unrelated to the iteration's changes. This cost one wasted release attempt (~25 min + ~30K tokens).

Decision Drivers

  • Release ceremony wall time should be proportional to change scope
  • Environment issues in unchanged layers should not block releases
  • Safety: changed layers MUST still run all gates
  • The --skip-unchanged-layers mechanism (FTR-443) already exists and is proven

Considered Options

  1. Auto-detect unchanged layers — release.sh always detects unchanged layers and skips their gates. No flag needed.
  2. Mandatory flag for non-C++ iterations — Require --skip-unchanged-layers as a process rule when no C++ changes exist.
  3. Status quo — Run all gates unconditionally, use --skip-unchanged-layers as an optional escape hatch.

Decision Outcome

Chosen option: "Auto-detect unchanged layers", because it eliminates both human error (forgetting the flag) and wasted compute. The detection mechanism is already implemented in FTR-443 (git diff --name-only against last tag).

Implementation

release.sh MUST auto-detect unchanged layers using git diff --name-only LAST_TAG..HEAD. Layer detection rules:

Layer File patterns Gates skipped when unchanged
C++ *.cpp, *.hpp, *.h, CMakeLists.txt, cmake/** C++ build, ctest, clang-tidy
Racket dslgen/**/*.rkt, dslgen/grammar/** Racket test suite
Database **/*.sql, database/** DB deploy, DB integration
Proto proto/**/*.proto gRPC generation

Override: --force-all-gates runs all gates regardless of detection. Use for release candidates or when environment validation is needed.

Positive Consequences

  • Eliminates environment-fragility release blockers for unchanged layers
  • Reduces DSL-only release ceremony from ~50 min to ~15 min
  • No human judgment needed (auto-detection vs. remembering a flag)

Negative Consequences

  • Risk of skipping a gate when a change is missed by the file pattern detection (mitigated: patterns are conservative and match broadly)
  • Must maintain the file pattern list as new layers are added

ADR: Build Acceleration — Unity Builds + Precompiled Headers for Generated Code

ADR: Build Acceleration — Unity Builds + Precompiled Headers for Generated Code

Date: 2026-02-17 Status: accepted Features: FTR-063, FTR-064 Supersedes: —

Context

Each bounded context generates many small C++ translation units (entity.cpp, command_repository.cpp, query_repository.cpp, command_service.cpp, query_service.cpp per entity). For fhir-health with 9 contexts, this means hundreds of compiler invocations. Each invocation pays the full include parsing cost for Qt, gRPC, Boost, and pqxx headers.

CMake 3.16+ provides built-in support for both unity builds (UNITY_BUILD target property) and precompiled headers (target_precompile_headers()). These require zero manual header batching — CMake handles everything.

Decision

KEEP both features, gated by CMake options defaulting to OFF:

  • MUSCOVITE_UNITY_BUILD — sets UNITY_BUILD ON + UNITY_BUILD_BATCH_SIZE 0 on all per-context library targets
  • MUSCOVITE_PCH — adds target_precompile_headers() with layer-appropriate headers

Both options are emitted into generated CMakeLists.txt files across all four layers (DBA, gRPC, Qt Widgets, QML).

Benchmark Results (2026-02-16, fhir-health, -j6)

Configuration Steps Wall Time Speedup
Baseline (no unity, no PCH) 914 1103s —
Unity build ON 216 183s 83.4%

Gate condition was "< 5% improvement = REVERT". 83.4% far exceeds the threshold.

PCH headers by layer:

  • DBA: <pqxx/pqxx>, <boost/uuid/uuid.hpp>, <boost/uuid/uuid_io.hpp>, <string>, <vector>, <optional>, <memory>, <stdexcept>
  • gRPC: <grpcpp/grpcpp.h>, <google/protobuf/util/json_util.h>
  • Qt Widgets: <QWidget>, <QLineEdit>, <QComboBox>, <QFormLayout>, <QVBoxLayout>
  • QML: <QObject>, <QString>, <QVariant>

Consequences

  • Clean builds of generated projects can be 6× faster with -DMUSCOVITE_UNITY_BUILD=ON
  • Zero impact when options are OFF (default) — no behavioral change for existing users
  • Unity builds may surface ODR violations if generated code uses static helper functions in .cpp files — audit periodically
  • ccache interaction: PCH may require CCACHE_SLOPPINESS=pch_defines,time_macros for optimal cache hit rates

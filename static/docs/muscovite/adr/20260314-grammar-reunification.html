ADR: Grammar Reunification — Merge Layout PEG into Canonical Grammar

ADR: Grammar Reunification — Merge Layout PEG into Canonical Grammar

Date: 2026-03-14 Status: accepted Deciders: Johannes Lochmann Supersedes: Partially supersedes ADR 20260223 (grammar separation only)

Context

ADR 20260223 separated the layout DSL grammar into muscovite-lay.peg (132 lines, 26 rules) to enforce concern separation between domain modeling (.ddd) and UI layout (.lay). In practice, 12 of the 26 rules were duplicated verbatim from muscovite.peg, with a source reference comment at muscovite-lay.peg:69-70.

This duplication created a silent drift risk: if MasterDirective, DetailDirective, or any shared clause evolved in muscovite.peg, the .lay grammar would diverge without warning until tests caught it. Additionally:

  1. The .lay grammar was small (26 rules, 132 lines) and stable since FTR-134.
  2. The lexer (lexer.rkt) already handled both vocabularies in one module.
  3. Grammar conformance tests validated both files, adding complexity for no gain.
  4. Layout directives reference aggregates and fields by name — semantically part of the domain vocabulary.

Decision

Merge all 21 layout-specific PEG rules from muscovite-lay.peg into muscovite.peg as section 9b (Layout Profile Grammar). Delete muscovite-lay.peg. Five lexical rules (IDENTIFIER, STRING, Spacing, EOL, EOF) were identical and already existed in the canonical grammar — they were not duplicated.

What Changes

  • muscovite.peg gains section 9b with 21 active layout profile rules
  • muscovite-lay.peg is deleted
  • Grammar conformance tests reference only the unified grammar
  • Grammar version bumped from 1.5.0 to 1.6.0

What Does NOT Change

  • .lay file extension remains a valid DSL file type
  • lay-parser.rkt remains a separate parser module
  • lay-types.rkt IR structs unchanged
  • project-parser.rkt PRESENTATION_PROFILE handling unchanged
  • generate.rkt orchestration flow unchanged
  • Parser tests (test-lay-parser.rkt) unchanged
  • FTR-134 enforcement (LAYOUT rejection in .ddd) unchanged

Consequences

  • Positive: Single source of truth for all PEG rules; no duplication drift risk
  • Positive: Simpler conformance test infrastructure (one grammar file)
  • Positive: Layout rules are now validated as part of the canonical grammar
  • Negative: Slightly larger muscovite.peg file (~40 lines net increase)

Metamorphic vs Sedimentary Facies - Distinct Concepts

Metamorphic vs Sedimentary Facies - Distinct Concepts

  • Status: accepted
  • Deciders: Domain modeling team
  • Date: 2026-01-07
  • Tags: petrology, stratigraphy, facies, classification

Context and Problem Statement

The term "facies" has two fundamentally different meanings in geology:

  1. Metamorphic facies: Pressure-Temperature (P-T) conditions indicator
  2. Sedimentary facies: Depositional environment indicator

These are completely different concepts that must not be conflated in the domain model.

Decision Drivers

  • Metamorphic facies (Eskola 1915, Turner 1948) are P-T based mineral assemblages
  • Sedimentary facies are depositional environment interpretations
  • Using same entity for both causes semantic confusion
  • Geologists expect clear distinction between these concepts

Decision Outcome

Maintain separate entities in appropriate contexts:

1. Metamorphic Facies (Petrology Context)

Located in petrology.ddd as MetamorphicFacies:

ENTITY MetamorphicFacies IDENTIFIED_BY id (
    id                    UUID        NOT NULL,
    name                  VARCHAR(50) NOT NULL,
    temperature_min_c     DECIMAL     NOT NULL,
    temperature_max_c     DECIMAL     NOT NULL,
    pressure_min_kbar     DECIMAL     NOT NULL,
    pressure_max_kbar     DECIMAL     NOT NULL,
    geothermal_gradient   VARCHAR(30),
    diagnostic_assemblages TEXT,
    typical_minerals      TEXT,
    tectonic_setting      TEXT,
    description           TEXT
)

Standard metamorphic facies (Eskola/Turner classification):

Facies T (°C) P (kbar) Setting Index Minerals
Zeolite 200-300 0.2-0.5 Burial Zeolites, prehnite
Prehnite-pumpellyite 250-350 0.3-0.7 Burial Prehnite, pumpellyite
Greenschist 300-500 0.3-0.8 Regional Chlorite, epidote, actinolite, albite
Blueschist 200-500 0.5-2.0 Subduction Glaucophane, lawsonite
Amphibolite 500-700 0.4-1.2 Regional Hornblende, plagioclase (>An17)
Granulite 700-900 0.7-1.2 Deep crust Pyroxene, garnet, sillimanite
Eclogite 450-700 1.2-3.0 Subduction Omphacite, pyrope garnet
Hornfels Variable Low Contact Andalusite, cordierite
Sanidinite >900 Low Xenolith Sanidine, mullite

2. Sedimentary Facies (Stratigraphy Context)

Located in stratigraphy.ddd as Facies (to be renamed SedimentaryFacies):

ENTITY SedimentaryFacies IDENTIFIED_BY id (
    id                        UUID        NOT NULL,
    facies_code               VARCHAR(10) NOT NULL,
    name                      VARCHAR(100) NOT NULL,
    facies_type               VARCHAR(30) NOT NULL,
    lithology                 VARCHAR(50) NOT NULL,
    grain_size                VARCHAR(20),
    sedimentary_structures    TEXT,
    fossils                   TEXT,
    depositional_process      VARCHAR(100),
    depositional_environment  VARCHAR(100),
    facies_associations       TEXT,
    description               TEXT
)
CHECK type_valid: facies_type IN ('lithofacies', 'biofacies', 'ichnofacies', 'seismic_facies')

Types of sedimentary facies:

Type Basis Example
Lithofacies Rock characteristics Trough cross-bedded sandstone (St)
Biofacies Fossil assemblage Coral-algal biofacies
Ichnofacies Trace fossils Skolithos ichnofacies
Seismic facies Seismic reflection patterns Parallel continuous reflectors

Walther's Law

For sedimentary facies, Walther's Law (1894) states:

"The vertical succession of facies reflects the lateral juxtaposition of depositional environments."

This applies ONLY to sedimentary facies, NOT metamorphic facies.

Key Differences Summary

Aspect Metamorphic Facies Sedimentary Facies
Basis P-T conditions Depositional environment
Defined by Mineral assemblage Lithology, fossils, structures
Context Petrology Stratigraphy
Example Greenschist facies Fluvial channel facies
Spatial P-T diagram Geographic distribution

Positive Consequences

  • Clear semantic separation prevents confusion
  • Each facies type in appropriate bounded context
  • Enables correct FK relationships
  • Matches geologist mental model

Negative Consequences

  • Two entities with similar names may initially confuse developers
  • Documentation must be clear about the distinction

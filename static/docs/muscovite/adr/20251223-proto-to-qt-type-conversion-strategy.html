Proto to Qt Type Conversion Strategy

Proto to Qt Type Conversion Strategy

  • Status: accepted
  • Deciders: jolo
  • Date: 2025-12-23
  • Tags: qt, frontend, protobuf, type-conversion

Technical Story: Defining consistent type mapping between Protocol Buffer types and Qt types for the frontend layer.

Context and Problem Statement

Generated gRPC code uses Protocol Buffer types (std::string, google::protobuf::Timestamp, etc.) which don't integrate well with Qt's type system (QString, QDateTime, Q_PROPERTY). We need a consistent strategy for converting between these type systems.

Decision Drivers

  • QML compatibility (Q_PROPERTY, QVariant)
  • QtWidgets compatibility (QAbstractItemModel)
  • Memory ownership clarity
  • Performance (avoid unnecessary copies)
  • Type safety (compile-time checks where possible)
  • Consistency across all generated code

Considered Options

  1. Wrapper Classes with Lazy Conversion - Wrap protobuf, convert on access
  2. Immediate Full Conversion - Convert entire message to Qt types
  3. Thin Wrapper with Direct Access - Expose protobuf fields, Qt helpers
  4. Code Generation with Qt Types - Generate Qt-native types from DSL

Decision Outcome

Chosen option: "Immediate Full Conversion", because it provides the cleanest Qt integration, clear memory ownership, and best QML compatibility. The performance cost is acceptable given our no-caching architecture (data is fetched per-use anyway).

Positive Consequences

  • Clean Qt types throughout UI layer
  • Simple memory ownership (Qt parent-child)
  • Perfect QML integration
  • No protobuf leakage to UI code
  • Type-safe Q_PROPERTY declarations

Negative Consequences

  • CPU cost of conversion on every fetch
  • Memory duplication (proto + Qt objects)
  • Generated code is larger

Implementation

TypeConverter Class (muscoviteqt)

namespace muscovite::qt {

class TypeConverter {
public:
    // Primitive conversions
    static QString toString(const std::string& s) {
        return QString::fromStdString(s);
    }
    
    static std::string fromString(const QString& s) {
        return s.toStdString();
    }
    
    static QByteArray toBytes(const std::string& s) {
        return QByteArray::fromStdString(s);
    }
    
    static std::string fromBytes(const QByteArray& b) {
        return b.toStdString();
    }
    
    // Timestamp conversions
    static QDateTime toDateTime(const google::protobuf::Timestamp& ts) {
        return QDateTime::fromSecsSinceEpoch(ts.seconds(), Qt::UTC)
            .addMSecs(ts.nanos() / 1000000);
    }
    
    static google::protobuf::Timestamp fromDateTime(const QDateTime& dt) {
        google::protobuf::Timestamp ts;
        ts.set_seconds(dt.toSecsSinceEpoch());
        ts.set_nanos((dt.time().msec()) * 1000000);
        return ts;
    }
    
    // UUID conversions (stored as string in proto)
    static QUuid toUuid(const std::string& s) {
        return QUuid::fromString(QString::fromStdString(s));
    }
    
    static std::string fromUuid(const QUuid& uuid) {
        return uuid.toString(QUuid::WithoutBraces).toStdString();
    }
};

} // namespace muscovite::qt

Generated Entity Wrapper Example

// Generated: MineralWrapper.hpp
class MineralWrapper : public QObject {
    Q_OBJECT
    Q_PROPERTY(QString id READ id CONSTANT)
    Q_PROPERTY(QString name READ name WRITE setName NOTIFY nameChanged)
    Q_PROPERTY(QString formula READ formula WRITE setFormula NOTIFY formulaChanged)
    Q_PROPERTY(double hardness READ hardness WRITE setHardness NOTIFY hardnessChanged)
    
public:
    // Construct from protobuf
    static MineralWrapper* fromProto(
        const geosciences::mineralogy::Mineral& proto, 
        QObject* parent = nullptr
    ) {
        auto wrapper = new MineralWrapper(parent);
        wrapper->id_ = TypeConverter::toString(proto.id());
        wrapper->name_ = TypeConverter::toString(proto.name());
        wrapper->formula_ = TypeConverter::toString(proto.formula());
        wrapper->hardness_ = proto.hardness();
        return wrapper;
    }
    
    // Convert back to protobuf
    geosciences::mineralogy::Mineral toProto() const {
        geosciences::mineralogy::Mineral proto;
        proto.set_id(TypeConverter::fromString(id_));
        proto.set_name(TypeConverter::fromString(name_));
        proto.set_formula(TypeConverter::fromString(formula_));
        proto.set_hardness(hardness_);
        return proto;
    }
    
    // Property accessors
    QString id() const { return id_; }
    QString name() const { return name_; }
    void setName(const QString& v) { 
        if (name_ != v) { name_ = v; emit nameChanged(); }
    }
    // ... other accessors
    
signals:
    void nameChanged();
    void formulaChanged();
    void hardnessChanged();
    
private:
    QString id_;
    QString name_;
    QString formula_;
    double hardness_;
};

List Model Integration

Generated models use converted types:

class MineralListModel : public QAbstractListModel {
    Q_OBJECT
    
public:
    enum Roles {
        IdRole = Qt::UserRole + 1,
        NameRole,
        FormulaRole,
        HardnessRole
    };
    
    QVariant data(const QModelIndex& index, int role) const override {
        auto* mineral = minerals_[index.row()];
        switch (role) {
            case IdRole: return mineral->id();
            case NameRole: return mineral->name();
            case FormulaRole: return mineral->formula();
            case HardnessRole: return mineral->hardness();
        }
        return {};
    }
    
private:
    QList<MineralWrapper*> minerals_;
};

AI Agent Guidance

When This Decision Applies

  • Generating Qt entity wrappers from DSL
  • Implementing service response handlers
  • Creating QML-compatible data objects

When This Decision Does NOT Apply

  • Pure gRPC layer (before Qt conversion)
  • Server-side code (uses protobuf directly)
  • Internal service-to-service communication

Enforcement

  • Generated code follows type mapping table exactly
  • No protobuf types leak past service layer
  • All Q_PROPERTY types are Qt-native

Key Files Affected

  • src/muscframework/qt/core/TypeConverter.hpp
  • generated-*/qt/entity/ - All entity wrappers
  • generated-*/qt/model/ - All list models

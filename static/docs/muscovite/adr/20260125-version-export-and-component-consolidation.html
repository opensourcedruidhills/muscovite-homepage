Version Export and Component Consolidation

Version Export and Component Consolidation

  • Status: accepted
  • Date: 2026-01-25
  • Tags: versioning, cmake, packaging, release

Context and Problem Statement

We need a single source of truth for versioning such that package version equals C++ version, with no exceptions. We also want to consolidate Qt into the main Muscovite package components and reuse the framework’s version module in sample projects.

Decision Drivers

  • Consistency: package version must equal C++ version
  • Reuse: samples should reuse the framework version module
  • Simplicity: single package with components (no separate Qt package)
  • Release automation: clear tag formats and prompt guidance

Decision Outcome

  1. Single source of truth: project(... VERSION X.Y.Z ...) is authoritative.
  2. Package version equals C++ version across all projects.
  3. Qt folded into muscovite components (no separate muscoviteqt package).
  4. Version header installed as part of the framework install.
  5. Version module exported via package config for reuse by samples.

Implementation Notes

  • Generate and install version.hpp from cmake/Version.cmake.
  • Export MUSCOVITE_CMAKE_MODULE_DIR and MUSCOVITE_VERSION in the package config.
  • Remove reliance on separate muscoviteqt package (use muscovite COMPONENTS qt).

Consequences

  • Consumers can rely on a single package with components.
  • Release artifacts are consistent: version header, package config, and tags agree.
  • Samples can reuse Version.cmake without copying.

2. Architecture & DSL Governance

Three-Layer Hierarchy

Layer Location Purpose
Layer 0 — Brain `dslgen/` (Racket) DSL parser, IR model, all generators
Layer 1 — Framework `framework/` (C++) Reusable base classes (Repository, UnitOfWork, errors)
Layer 2 — Generated Consumer repos via `muscovite-cpp` Generated from `.ddd` models. Consumer repos own their lifecycle.

DSL Governance Chain

PEG Grammar (spec) → Parser (impl) → Grammar Conformance Test (parity check)
       ↓                    ↓                        ↓
  Syntax rules      IR struct instances      Keyword drift detection
       ↓                    ↓                        ↓
  doc/DSL.md (auto)   Generators (emit)       doc/CONTEXT.md (metrics)

Rule: Grammar changes precede parser changes (ADR 20260211). Zero opaque stubs.

Theme: Close the Coordinator

First iteration executed end-to-end through the NATS-based multi-agent coordinator. TLA+ formal verification passed: 242,662,977 states, 8 safety invariants, zero violations.

Added

  • (FTR-398): OpenTelemetry agent session observability
    • OTEL collector sidecar configuration (OTLP gRPC/HTTP receivers)
    • NATS telemetry bridge (agents/telemetry_bridge.py) for span aggregation
    • Coordinator CLI telemetry commands: status, watch, alert
    • coord.telemetry.* NATS subjects for real-time worker health monitoring
  • (FTR-400): Hook-based safety gates for agent sessions
    • Role-based Copilot CLI hooks (.copilot/hooks/safety-gates.js)
    • Force-push, DROP DATABASE, out-of-scope edit detection
    • Leader: ask | QA: ask on merge | Worker: deny risky ops | Planner: deny writes
    • NATS hook event logging (coord.hook.* subjects)
  • (FTR-401): End-to-end coordinator dogfood run
    • 72 NATS messages captured across full FTR lifecycle
    • Worker assignment via JetStream push, worktree isolation verified
    • Role enforcement validated: leader review rejected, QA approval required
    • State machine: pending → assigned → progressing → pr_ready → reviewing → merged

TLA+ Verification

Coordinator model checked with TLC: 3 FTRs, 2 workers, depth 27. 242M states generated (6.57M distinct), 55 min on 12 cores. 8 invariants verified: TypeInvariant, OneWorkerOneFtr, ExclusiveTagMutex, GovernorLimits, MergeSingleThread, LocksConsistent, WorkerFtrBidir, DeadWorkersClean.

  • NATS client-side timeouts: server processes commands but replies are slow
  • No per-FTR merge command: amend-state used as workaround
  • DEPENDENCIES.toml [graph] format undocumented

1. System Purpose

2. Architecture & DSL Governance

Three-Layer Hierarchy

Layer Location Purpose
Layer 0 — Brain `dslgen/` (Racket) DSL parser, IR model, all generators
Layer 1 — Framework `framework/` (C++) Reusable base classes (Repository, UnitOfWork, errors)
Layer 2 — Generated Consumer repos via `muscovite-cpp` Generated from `.ddd` models. Consumer repos own their lifecycle.

DSL Governance Chain

PEG Grammar (spec) → Parser (impl) → Grammar Conformance Test (parity check)
       ↓                    ↓                        ↓
  Syntax rules      IR struct instances      Keyword drift detection
       ↓                    ↓                        ↓
  doc/DSL.md (auto)   Generators (emit)       doc/CONTEXT.md (metrics)

Rule: Grammar changes precede parser changes (ADR 20260211). Zero opaque stubs.

[1.75.0] -- 2026-03-22

Added

  • FTR-491: Idempotency keys — proto fields and SQL schema

    • Added IDEMPOTENCY_TTL "<duration>" DSL keyword to project.mus (default: 24h)
    • Added optional string idempotency_key to all gRPC write request messages (Create/Update/Delete for aggregate roots; Add/Update/Remove for child entities)
    • Generated {schema}.{aggregate}_idempotency_log table per aggregate root with deep-verify SQL (DO $$ ASSERT blocks) and TTL index on created_at

  • FTR-492: Idempotency keys — C++ service implementation

    • Generated gRPC service methods now check {schema}.{aggregate}_idempotency_log before performing write operations (create/update/delete and child entity ops)
    • Duplicate key returns ALREADY_EXISTS status; successful operations register the key with ON CONFLICT DO NOTHING for race safety

  • FTR-493: Idempotency keys — Qt client stub UUID generation

    • Qt gRPC client .cpp generator now emits #include <QUuid> and auto-generates a fresh QUuid::createUuid() idempotency key for every write RPC call

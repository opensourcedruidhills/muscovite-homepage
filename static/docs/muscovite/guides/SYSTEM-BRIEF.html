2. Architecture & DSL Governance

Three-Layer Hierarchy

Layer Location Purpose
Layer 0 — Brain `toolchain/muscomp/` (C++) DSL parser, IR model, all generators
Layer 1 — Framework `framework/` (C++) Reusable base classes (Repository, UnitOfWork, errors)
Layer 2 — Generated Consumer repos via `muscovite-cpp` Generated from `.ddd` models. Consumer repos own their lifecycle.

DSL Governance Chain

PEG Grammar (spec) → Parser (impl) → Grammar Conformance Test (parity check)
       ↓                    ↓                        ↓
  Syntax rules      IR struct instances      Keyword drift detection
       ↓                    ↓                        ↓
  doc/DSL.md (auto)   Generators (emit)       doc/CONTEXT.md (metrics)

Rule: Grammar changes precede parser changes (ADR 20260211). Zero opaque stubs.

⚠️ BREAKING: Racket Retirement

Muscovite v2.9.0 removes the Racket generator pipeline entirely. The dslgen/ directory (~350 files, ~177K lines) has been deleted. All generation is now handled by the native C++ compiler muscomp. Racket is no longer a build dependency.

Added

  • FTR-687: Hardened stdlib — _GLIBCXX_ASSERTIONS enabled project-wide via cmake/HardenedStdlib.cmake and all Conan profiles. Catches undefined behavior (out-of-bounds, invalid iterators) at runtime in debug and release builds.
  • FTR-684: CMake grammar freshness — CMAKE_CONFIGURE_DEPENDS on muscovite.peg ensures muscomp automatically reconfigures when the PEG grammar changes. Eliminates the stale-grammar bug from v2.6.2.

Changed

  • FTR-688: All scripts and quality gates switched from Racket to muscomp:
    • release.sh: Racket gates → muscomp build + tests + DSL lint
    • agent-pr.sh: Racket gates → muscomp equivalents
    • lint-ddd.sh: rewritten to use muscomp --lint-only
    • collect-release-metrics.sh: racket layers → muscomp layers
    • generate-dsl-reference.sh: muscomp --generate-dsl-reference
    • validate-examples.sh, generate-audience-reports.sh, run-tlaplus-check.sh
    • update-readme-metrics.sh, generate-system-brief.sh, metrics-trend.sh
    • All CMake modules (DatabaseProject, Documentation, ExternalExamples, etc.)

Removed

  • FTR-689: Deleted dslgen/ — entire Racket codebase (~177K lines)
    • Grammar relocated: dslgen/grammar/muscovite.peg → grammar/muscovite.peg
    • doc/RACKET-PATTERNS.md archived to doc/archive/
    • Racket test files, tools, and bytecode compilation removed
  • FTR-690: Purged Racket references from 37 documentation files, 4 prompt files deleted, all AI agent commands updated. ADRs preserved as historical records.
  • tools/collect-racket-coverage.sh (no longer applicable)

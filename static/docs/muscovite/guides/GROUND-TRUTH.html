Muscovite Ground Truth Reference

Muscovite Ground Truth Reference

Purpose: Canonical reference for AI agents to prevent hallucinations. Documents the three-layer architecture, exact Racket structures, naming conventions, and type mappings.

0. Three-Layer Architecture (Non-Negotiable)

Muscovite enforces a strict three-layer hierarchy. Every change must respect this separation.

Layer Name Location Purpose
Layer 0 The Brain (Generator/IR) dslgen/ (Racket) Defines the DSL IR, owns all templates and code generation
Layer 1 The Framework framework/ (C++ base libraries) Reusable base infrastructure (Repository, UnitOfWork, errors, connections, messaging). No domain logic.
Layer 2 Generated Applications examples/*/generated/ 100% generated from .ddd models. Never manually edited.

Lockstep Development (Mandatory)

When adding a DSL feature (Layer 0), you must:

  1. Update Layer 1 framework if new base behavior is needed
  2. Update Layer 0 templates to generate Layer 2 code that uses it
  3. Regenerate samples (Layer 2)

Layer Mapping: IR → Framework → Generated Code

IR Struct Layer 1 Base Layer 2 Generated
entity Repository base (framework/dba/.../repository.hpp) Entity struct (e.g., patient.hpp)
aggregate UnitOfWork (framework/dba/.../unit_of_work.hpp), ValidationErrors (framework/validation/.../validation_errors.hpp) *CommandService / *QueryService / *DomainService classes
repository RepositoryBase *CommandRepository / *QueryRepository
domain-event DomainEvent struct in UnitOfWork, Publisher (framework/messaging/.../publisher.hpp) domain_events table + emission in services
sourced-event (no framework base) Generated EventStore per aggregate
state-machine (no framework base) *StateMachine C++ class, state lookup tables, transition log, gRPC RPCs, Qt state-aware UI
entity + display-format (no framework base) QML ViewModels with Q_PROPERTY

I. Core Racket Structures (dslgen/model/ddd-types.rkt)

1. Bounded Context

(struct bounded-context
  (uuid          ; String: stable UUID for semantic diff
   name           ; Symbol: context name
   sqitch-project ; String or #f: override sqitch project name
   builtin?       ; Boolean: #t if framework-provided (Security, Spatial)
   aggregates     ; List of aggregate
   entities       ; List of entity: standalone entities NOT in aggregates
   repositories   ; List of repository
   services       ; List of domain-service
   events         ; List of domain-event (messaging/side effects)
   sourced-events ; List of sourced-event (event sourcing)
   specifications ; List of specification
   policies       ; List of policy
   sagas          ; List of saga
   state-machines ; List of state-machine: entity lifecycle graphs (ADR 20260221)
   lookup-tables  ; List of lookup-table
   value-objects  ; List of value-object
   strong-types   ; List of strong-type (PostgreSQL DOMAIN)
   projections    ; List of projection (CQRS read models)
   context-defaults ; List of context-default
   grounding      ; grounding or #f: AI context anchoring
   doc)           ; String or #f: documentation
  #:transparent)

Constructor:

(make-bounded-context name
  #:uuid [uuid (generate-uuid)]
  #:aggregates [aggregates '()]
  #:entities [entities '()]
  ...)

2. Entity

(struct entity
  (uuid         ; String: stable UUID for semantic diff
   name         ; Symbol: entity name
   id-field     ; field: the identity field (always named 'id)
   fields       ; List of field: non-id fields (own fields only)
   invariants   ; List of invariant: CHECK constraints
   commands     ; List of command
   doc          ; String or #f: documentation
   synonyms     ; List of String: alternative names
   parent-name  ; Symbol or #f: parent entity for TPT inheritance
   metadata     ; ent-metadata: relationships, indexes, hooks
   seed         ; seed-data or #f: optional test data
   seed-import  ; seed-import or #f: CSV/JSON import
   display-formats ; List of display-format: UI display patterns
   uses-value-objects  ; List of uses-value-object: @inline mixins
   has-value-objects   ; List of has-value-object: @owned/@shared refs
   instructions        ; List of instruct-spec: embedded AI prompts
   concurrency         ; concurrency-config or #f: locking strategy (ADR 20260215)
   bitemporal          ; bitemporal-config or #f: temporal tracking strategy (FTR-107)
   source)             ; String or #f: external reference traceability (@SOURCE, FTR-087)
  #:transparent)

Key invariants:

  • id-field is always first, always named id, always UUID
  • fields contains only own fields (no inherited fields)
  • TPT: Use parent-name to link child → parent

3. Field Constraint (FTR-261)

(struct field-constraint
  (kind       ; Symbol: 'min, 'max, 'not-empty, 'not-in-future, 'not-in-past, 'pattern
   value      ; Number, String, or #f: constraint argument
   relative?) ; Boolean: #f = per-field, #t = cross-field (FTR-262)
  #:transparent)

Kind values: 'min (numeric lower bound), 'max (numeric upper bound), 'not-empty (string/collection guard), 'not-in-future (temporal upper bound), 'not-in-past (temporal lower bound), 'pattern (regex validation).

4. Field

(struct field
  (uuid        ; String: stable UUID for rename tracking
   name        ; Symbol: field name (snake_case)
   type        ; type-spec: the type
   constraints ; List of field-constraint structs (FTR-261)
   doc         ; String or #f: field documentation
   required?   ; Boolean: from @required (NOT NULL)
   unique?     ; Boolean: from @unique (UNIQUE constraint)
   spatial-queryable? ; Boolean: generates spatial query RPCs
   searchable? ; Boolean: from @searchable (OpenSearch index, FTR-112)

Constructor:

(make-field name type-name 
  #:uuid [uuid (generate-uuid)]
  #:required? [required? #f]
  #:unique? [unique? #f]
  #:spatial-queryable? [spatial-queryable? #f]
  #:searchable? [searchable? #f]
  ...)

6. Relationship

(struct relationship
  (uuid         ; String: stable UUID
   name         ; Symbol: relationship name (role)
   direction    ; Symbol: 'to-one | 'to-many | 'many-to-many
   target-entity; Symbol: target entity name
   target-context ; Symbol or #f: qualified refs like Basedata.Status
   required?    ; Boolean: NOT NULL FK (REQUIRED vs OPTIONAL)
   multiplicity ; Symbol: 'one or 'many

Critical rule: Relationships are explicit-only (ADR 20260106). No inference from _id naming.

8. Value Object

(struct value-object
  (uuid           ; String: stable UUID
   name           ; Symbol: value object name
   fields         ; List of field
   invariants     ; List of invariant
   doc            ; String or #f
   synonyms       ; List of String
   storage-mode   ; Symbol: 'inline | 'owned | 'shared
   computed-fields; List of computed-field
   immutable-fields; List of Symbol: preserved on UPDATE

Storage modes (ADR 20260117):

  • 'inline: Fields flattened into parent table (mixin)
  • 'owned: Child table with CASCADE delete
  • 'shared: Dimension/lookup table (star schema)

(struct strong-type

Examples: Email, Phone, PostalCode, PositiveDecimal, Latitude, Longitude

16. State Machine (ADR 20260221)

(struct state-machine
  (uuid            ; String: stable UUID for semantic diff
   name            ; Symbol: state machine name (e.g., 'OrderLifecycle)
   for-entity      ; Symbol: target entity name (e.g., 'Order)
   states          ; List of sm-state: all declared states
   transitions     ; List of sm-transition: all directed edges
   initial-state   ; Symbol: the initial state name
   terminal-states ; List of Symbol: terminal state names
   enforce-at-db?  ; Boolean: #t if ENFORCE_AT DB_LEVEL (generates guard trigger, FTR-126)
   doc)            ; String or #f: documentation
  #:transparent)

Constructor: (make-state-machine name for-entity #:uuid [...] #:states ['()] #:transitions ['()] #:initial-state [#f] #:terminal-states ['()] #:enforce-at-db? [#f] #:doc [#f])

Key invariants:

  • Context-level construct (like Saga), not entity-inline
  • for-entity must reference an entity in the same context
  • initial-state must name exactly one state
  • terminal-states must name at least one state
  • Validation enforces: reachability, no orphan transitions, determinism (no duplicate from+event pairs)
  • When enforce-at-db? is #t, generates: {entity}_allowed_transition table, PL/pgSQL validation function, BEFORE UPDATE trigger, and pgTAP test file

16a. SM State

(struct sm-state
  (name          ; Symbol: state name (e.g., 'Draft, 'Submitted)
   entry-action  ; Symbol or #f: action invoked on state entry
   exit-action   ; Symbol or #f: action invoked on state exit
   doc)          ; String or #f: documentation
  #:transparent)

Constructor: (make-sm-state name #:entry-action [#f] #:exit-action [#f] #:doc [#f])

16b. SM Transition

(struct sm-transition
  (from-state    ; Symbol: source state name
   to-state      ; Symbol: target state name
   event         ; Symbol: triggering event name
   guard         ; String or #f: guard expression (SQL-compatible)
   action        ; Symbol or #f: action invoked on transition
   doc)          ; String or #f: documentation
  #:transparent)

Constructor: (make-sm-transition from-state to-state event #:guard [#f] #:action [#f] #:doc [#f])

Generated artifacts per state machine: | Layer | Artifact | |-------|----------| | PostgreSQL | {entity}_state lookup table + current_state_id FK on entity + {entity}_state_transition_log table | | PostgreSQL (ENFORCE_AT DB_LEVEL) | {entity}_allowed_transition table + {entity}_validate_transition() PL/pgSQL function + BEFORE UPDATE trigger (FTR-126) | | C++ Lookup | lookup::{entity}_state::* compile-time UUIDv5 constants | | C++ DBA | {Entity}StateMachine class with can_transition(), transition(), allowed_transitions() | | gRPC | {Entity}StateMachineService with TransitionState, GetAllowedTransitions, GetStateTransitionLog RPCs | | Qt Widgets | {Entity}StateWidget with state display + event buttons | | QML | {Entity}StateViewModel + {Entity}StatePanel.qml | | Docs | Mermaid stateDiagram-v2 (standalone .mmd + inline AsciiDoc) | | pgTAP (ENFORCE_AT DB_LEVEL) | {entity}_state_machine_guard_test.sql — runtime trigger validation (FTR-126) |

17. Bitemporal Config (FTR-107, ADR 20260221)

(struct bitemporal-config
  (valid-from-field  ; Symbol: name of valid-time start column (e.g., 'valid_from)
   valid-to-field    ; Symbol: name of valid-time end column (e.g., 'valid_to)
   system-versioned? ; Boolean: #t if SYSTEM_TIME AUTO is specified
   granularity)      ; Symbol: 'second | 'millisecond | 'microsecond (default 'microsecond)
  #:transparent)

Constructor: (make-bitemporal-config valid-from valid-to #:system-versioned? [#f] #:granularity ['microsecond])

Entity field: entity struct has bitemporal field — bitemporal-config or #f.

Key invariants:

  • Entity-level inline clause (like CONCURRENCY), not context-level
  • valid-from-field and valid-to-field must not conflict with other entity field names
  • System-versioning (SYSTEM_TIME AUTO) adds system_from + system_to TIMESTAMPTZ columns
  • Without system-versioning, only valid-time columns are generated

Generated artifacts per bitemporal entity: | Layer | Artifact | |-------|----------| | PostgreSQL | {valid_from}, {valid_to} TIMESTAMPTZ columns (+ system_from, system_to if system-versioned) | | PostgreSQL | btree_gist extension, EXCLUDE USING GIST (id WITH =, tstzrange({valid_from}, {valid_to}) WITH &&) constraint | | PostgreSQL | System-time trigger: system_from = NOW() on INSERT, close+reopen on UPDATE | | C++ DBA | find_as_of(id, valid_at) temporal point query (valid-time lookup) | | C++ DBA | find_as_of(id, valid_at, system_at) bitemporal point query (system-versioned only) | | C++ DBA | find_history(id) full valid-time history | | C++ DBA | find_corrections(id) audit trail via system time (system-versioned only) | | gRPC | valid_at + system_at optional Timestamp fields on GetXxxRequest | | gRPC | GetHistory{Xxx}Request/Response messages + GetHistoryXxx RPC | | Qt QML ViewModel | Q_PROPERTY(QDateTime validAt) + Q_PROPERTY(QDateTime systemAt) (system-versioned only) |

18. Expression AST (FTR-284, ADR 20260307 Tier 1)

Expression nodes for CHECK constraints, guards, and lifecycle hooks. 9 base nodes + 5 collection predicate nodes.

Base expression nodes (ddd-types.rkt:807-852):

Struct Fields Purpose
binary-expr op left right Binary: =, !=, <, >, AND, OR, +, -, *, /, matches
unary-expr op operand Unary: not, -
field-ref entity field Field reference: entity.field or field (entity=#f)
literal-expr value Literal: number, string, boolean, 'null
match-expr field pattern Pattern match: field MATCHES "regex"
call-expr func args Function call: func(args...)
null-check-expr field negated? IS NULL / IS NOT NULL
in-expr field values field IN (val1, val2, ...)
between-expr field low high field BETWEEN low AND high

Collection predicate nodes (ddd-types.rkt:854-886):

Struct Fields Purpose
not-in-expr field values field NOT IN (val1, val2, ...)
empty-check-expr field negated? field IS EMPTY / field IS NOT EMPTY
set-op-expr op left right Set ops: 'intersects, 'subset-of, 'contains-all
contains-expr list-expr value-expr CONTAINS(list-field, value)
list-expr items List literal: (v1, v2, ...) or [v1, v2, ...]

Accessor/Mutator Patterns (gRPC)

Reading from proto:

(define (type->proto-accessor-pattern type-sym)
  (case type-sym
    [(String Integer Boolean) 'direct]          ; req.field_name()
    [(UUID Decimal Email Phone) 'value]         ; req.field_name().value()
    [(DateTime Timestamp) 'seconds]             ; req.field_name().seconds()
    [(Date) 'date-fields]                       ; req.field_name().year(), .month(), .day()
    [(Point LineString Polygon) 'wkt]))         ; wkt_to_spatial(req.field_name())

Writing to proto:

(define (type->proto-mutator-pattern type-sym)
  (case type-sym
    [(String Integer Boolean) 'set]             ; req.set_field_name(val)
    [(UUID Decimal Email Phone) 'mutable-value] ; req.mutable_field_name()->set_value(val)
    [(DateTime Timestamp) 'mutable-seconds]     ; req.mutable_field_name()->set_seconds(val)
    [(Date) 'mutable-date]))                    ; set year/month/day

Given context PatientContext:

CREATE SCHEMA patient;              -- Domain schema

CREATE TABLE patient.patient (...); -- Entity table

ID Fields (UUIDv7)

Always:

  • Named id
  • Type UUID
  • Primary key
  • Default gen_random_uuid() or uuidv7() (if available)
CREATE TABLE patient.patient (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    ...
);

Foreign Keys

NO inference from _id naming (ADR 20260106). Explicit RELATIONSHIPS block required.

ENTITY Encounter (...
    patient_id UUID NOT_NULL  -- Just a UUID field, NOT auto-FK
)
RELATIONSHIPS {
    patient -> Patient REQUIRED ONE;  -- Explicit FK declaration
}

Generated:

ALTER TABLE encounter.encounter
    ADD CONSTRAINT fk_encounter_patient
    FOREIGN KEY (patient_id) REFERENCES patient.patient(id);

CHECK value_arc:

V. Domain Services as Aggregate Boundary (ADR 20260120)

MANDATORY Architecture

All aggregate access through domain services. No stored procedures.

gRPC → DomainService → [Tier1: try_from] → [Tier2: EntityValidator] → [Tier3: PluginValidator] → CommandService → CommandRepository (pqxx) → PostgreSQL
                                                                                                       └── emit event to domain_events table
gRPC → DomainService → QueryService → QueryRepository (pqxx) → PostgreSQL  (queries bypass validation)

Generated Services

DomainService (orchestration — FTR-270, ADR 20260307):

class PatientDomainService final {
public:
    explicit PatientDomainService(
        PatientCommandService& command_service,
        PatientQueryService& query_service
    );

    // Command operations (three-tier validation pipeline)
    [[nodiscard]] auto create(Patient&& entity)
        -> std::expected<Patient, muscovite::validation::ValidationErrors>;
    [[nodiscard]] auto update(Patient&& entity)
        -> std::expected<Patient, muscovite::validation::ValidationErrors>;
    [[nodiscard]] auto remove(const boost::uuids::uuid& id)
        -> std::expected<bool, muscovite::validation::ValidationErrors>;

    // Query operations (passthrough, no validation)
    [[nodiscard]] auto find_by_id(const boost::uuids::uuid& id)
        -> DbResult<std::optional<Patient>>;
    [[nodiscard]] auto find_all()
        -> DbResult<std::vector<Patient>>;

private:
    PatientCommandService& command_service_;
    PatientQueryService& query_service_;
};

CommandService (writes):

class PatientCommandService {
public:
    PatientCommandService(
        PatientCommandRepository& patient_repo,
        EncounterCommandRepository& encounter_repo,  // children
        ConnectionPool& pool
    );

    // Root operations
    Patient create(const CreatePatientRequest& req);
    Patient update(const UpdatePatientRequest& req);
    void remove(const boost::uuids::uuid& id);

    // Child operations (through root)
    Encounter add_encounter(const boost::uuids::uuid& patient_id, 
                           const AddEncounterRequest& req);
    
private:
    // Single transaction per command
    template<typename F>
    auto execute(F&& func) {
        return muscovite::dba::execute(pool_, std::forward<F>(func));
    }
};

QueryService (reads):

class PatientQueryService {

VI. CQRS Event Emission (ADR 20260120)

Every aggregate gets a {schema}.domain_events table (id, aggregate_type, aggregate_id, event_type, payload TEXT, occurred_at, version). Events emitted inline in CommandService within the same transaction. The payload column stores pre-serialized JSON from protobuf MessageToJsonString() (ADR 20260306).

VII. Event Sourcing (ADR 20260204)

Opt-in per aggregate with EVENT_SOURCED. Typed event tables (no JSON), per-event payload tables, snapshot tables. Dual-write strategy: single transaction appends typed event + updates snapshot + updates entity table. No replay needed (snapshot = current state).

DSL: SOURCED_EVENT Name @uuid "..." (fields) APPLIES_TO AggregateRoot

X. INSTRUCT Blocks (MEP - Muscovite Embedded Prompts)

Platform-Agnostic AI Guidance

Per ADR 20260205: INSTRUCT blocks embed domain-specific AI prompts in the DSL.

(struct instruct-spec
  (uuid      ; String: stable UUID
   name      ; Symbol: instruction name
   intent    ; String: natural language intent
   inputs    ; List of string: inputs
   output    ; Symbol or #f: output type/code
   strategy  ; Symbol: 'deterministic, etc.
   doc)      ; String or #f
  #:transparent)

DSL Syntax (Proposed)

ENTITY Patient (...) {
    INSTRUCT calculate_bmi @uuid "..." (
        "Calculate Body Mass Index from height and weight"
        INPUTS: height_cm, weight_kg
        OUTPUT: Decimal
        STRATEGY: deterministic
    )
    DOC "Derived metric for clinical assessment";
}

Multi-Target Generation

INSTRUCT blocks generate:

  1. C++ method stubs with intent comment
  2. SQL CHECK constraints (if deterministic)
  3. QML validators for client-side logic

Key constraint: Must be platform-agnostic (no C++-specific or SQL-specific logic in DSL).

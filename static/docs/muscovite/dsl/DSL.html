18b. State Machine (Entity Lifecycle — ADR 20260221) Context-level construct for formal intra-entity lifecycle modeling. Generates state lookup tables, transition log, C++ state machine class, gRPC transition RPCs, and Qt state-aware UI. Example: STATE_MACHINE OrderLifecycle @uuid "..." 34. Anti-Corruption Layer

35a. MAPPING Block (Brownfield Migration Factory — FTR-080, ADR 20260224-002) MAPPING blocks define field-level transformations between legacy schema structures and Muscovite target entities. Used by the migration factory generators to produce type-safe extractor and transformer components. Example: MAPPING LegacyCustomer @uuid "..." { FROM "legacy_customers" INTO Customer; FIELD customer_id -> id COPY_UUID; FIELD fname -> given_name @required COPY; FIELD lname -> family_name @required COPY; FIELD customer_email -> email @unique VALIDATE_EMAIL; FIELD birth_dt -> birth_date COERCE_DATE format="MM/dd/yyyy"; FIELD is_active -> active DEFAULT true COPY_BOOL; FIELD created_ts -> created_at COERCE_TIMESTAMP; DOC "Maps legacy customer table to Patient entity"; } 35b. LEGACY_VIEW (Brownfield Extraction — FTR-427, ADR 20260313) LEGACY_VIEW captures the definition of a database view discovered during brownfield extraction. The BODY clause holds the raw SQL definition as a triple-quoted string. SOURCE_TABLES lists the tables the view reads from. CALLS lists stored procedures or functions the view references. Example: LEGACY_VIEW ActivePatients @uuid "..." { SOURCE_TABLES patients, appointments; MATERIALIZED false; DOC "Patients with appointments in the last 30 days"; BODY """ SELECT p.* FROM patients p JOIN appointments a ON a.patient_id = p.id WHERE a.date > NOW() - INTERVAL '30 days' """; } 35c. LEGACY_PROCEDURE (Brownfield Extraction — FTR-427, ADR 20260313) LEGACY_PROCEDURE captures the definition of a stored procedure or function discovered during brownfield extraction. PARAMS declares typed parameters with direction (IN/OUT/INOUT). CLASSIFICATION categorizes the procedure for migration planning (e.g., 'data_access', 'business_logic', 'utility'). Example: LEGACY_PROCEDURE CalculateBalance @uuid "..." { PARAMS ( account_id INTEGER IN, balance DECIMAL(19,4) OUT ); RETURNS DECIMAL(19,4); REFERENCES accounts, transactions; CALLS AuditLog; CLASSIFICATION "business_logic"; DOC "Calculate current balance from transaction history"; BODY """ SELECT SUM(amount) INTO balance FROM transactions WHERE account_id = $1 """; } 35d. FOREIGN CONTEXT (Anti-Corruption Layer — FTR-144, ADR 20260223) Typed ACL constructs for interfacing with external systems. FOREIGN_TYPE declares the shape of external data structures. TRANSLATE rules define declarative field-level mappings. The TRANSLATE sub-grammar is intentionally restricted: no IF/CASE/WHEN, no lambdas, no function calls (except type casts via ::TYPE). Complex logic belongs in Layer 2 Sidecar files. Example: FOREIGN CONTEXT PaymentGateway @uuid "..." { FOREIGN_TYPE PaymentResponse @uuid "..." ( transaction_id TEXT NOT_NULL @uuid "...", amount_cents INTEGER NOT_NULL @uuid "...", currency_code TEXT NOT_NULL @uuid "...", status_text TEXT NOT_NULL @uuid "..." ) DOC "External payment gateway response"; TRANSLATE Payment FROM PaymentResponse @uuid "..." { id <- transaction_id::UUID, amount <- amount_cents / 100, currency <- currency_code, is_approved <- status_text == "APPROVED" }; }

Grammar Rules

1. Constraint for AI generation — LLMs follow this grammar to produce

syntactically valid .ddd files

2. Specification for parser evolution — grammar changes drive parser

changes (not the reverse)

3. Living documentation — the grammar IS the DSL reference

Notation: 'KEYWORD' Literal keyword (case-insensitive in lexer, uppercase canonical) Non-terminal reference e1 / e2 Ordered choice (try e1 first) e* Zero or more e+ One or more e? Optional !e Not-predicate (negative lookahead) &e And-predicate (positive lookahead) ( ... ) Grouping

Comment

Lexical conventions:

  • Comments: '--' to end-of-line, '/* ... */' block
  • Strings: double-quoted "..." or single-quoted '...' with escapes
  • Numbers: integer or decimal (e.g., 42, 3.14, -89)
  • Identifiers: [a-zA-Z_][a-zA-Z0-9_-]*
  • Keywords: case-insensitive (CONTEXT = context = Context)
  • Annotations: '@' prefix (e.g., @uuid, @required, @unique)
  • Operators: ->, <-, <->, =, !=, <, <=, >, >=, ~, &&, ||
  • Punctuation: ( ) { } [ ] , ; . : @ ?
  • Variable refs: $identifier (in policy/saga parameter expressions)

35a. MAPPING Block (Brownfield Migration Factory — FTR-080, ADR 20260224-002)

MAPPING blocks define field-level transformations between legacy schema structures and Muscovite target entities. Used by the migration factory generators to produce type-safe extractor and transformer components. Example: MAPPING LegacyCustomer @uuid "..." { FROM "legacy_customers" INTO Customer;

35b. LEGACY_VIEW (Brownfield Extraction — FTR-427, ADR 20260313)

LEGACY_VIEW captures the definition of a database view discovered during brownfield extraction. The BODY clause holds the raw SQL definition as a triple-quoted string. SOURCE_TABLES lists the tables the view reads from. CALLS lists stored procedures or functions the view references. Example: LEGACY_VIEW ActivePatients @uuid "..." { SOURCE_TABLES patients, appointments; MATERIALIZED false; DOC "Patients with appointments in the last 30 days"; BODY """ SELECT p.* FROM patients p JOIN appointments a ON a.patient_id = p.id WHERE a.date > NOW() - INTERVAL '30 days' """; }

LegacyView <- 'LEGACY_VIEW' IDENTIFIER UuidAnnotation
LegacyViewBody <- LegacyViewClause*
LegacyViewClause <- LegacyViewSourceTables
LegacyViewSourceTables <- 'SOURCE_TABLES' IdentifierList ';'
LegacyViewCalls <- 'CALLS' IdentifierList ';'
LegacyViewMaterialized <- 'MATERIALIZED' BooleanLiteral ';'

35c. LEGACY_PROCEDURE (Brownfield Extraction — FTR-427, ADR 20260313)

35d. FOREIGN CONTEXT (Anti-Corruption Layer — FTR-144, ADR 20260223)

Typed ACL constructs for interfacing with external systems. FOREIGN_TYPE declares the shape of external data structures. TRANSLATE rules define declarative field-level mappings. The TRANSLATE sub-grammar is intentionally restricted: no IF/CASE/WHEN, no lambdas, no function calls (except type casts via ::TYPE). Complex logic belongs in Layer 2 Sidecar files. Example: FOREIGN CONTEXT PaymentGateway @uuid "..." { FOREIGN_TYPE PaymentResponse @uuid "..." ( transaction_id TEXT NOT_NULL @uuid "...", amount_cents INTEGER NOT_NULL @uuid "...", currency_code TEXT NOT_NULL @uuid "...", status_text TEXT NOT_NULL @uuid "..." ) DOC "External payment gateway response"; TRANSLATE Payment FROM PaymentResponse @uuid "..." { id <- transaction_id::UUID, amount <- amount_cents / 100, currency <- currency_code, is_approved <- status_text == "APPROVED" }; }

ForeignContext <- 'FOREIGN' 'CONTEXT' IDENTIFIER UuidAnnotation
ForeignContextItem <- ForeignType / TranslateRule / DocClause
ForeignType <- 'FOREIGN_TYPE' IDENTIFIER UuidAnnotation
TranslateRule <- 'TRANSLATE' QualifiedIdentifier 'FROM' QualifiedIdentifier
FieldMappingList <- FieldMapping ( ',' FieldMapping )* ','?
FieldMapping <- IDENTIFIER '<-' TranslateExpr
TranslateExpr <- TranslateArith
TranslateCast <- QualifiedIdentifier '::' IDENTIFIER
TranslateArith <- QualifiedIdentifier ( '+' / '-' / '*' / '/' ) ( NUMBER / QualifiedIdentifier )
TranslateComparison <- QualifiedIdentifier '==' STRING
TranslateConcat <- QualifiedIdentifier '||' ( STRING / QualifiedIdentifier )
TranslateNull <- QualifiedIdentifier '??' ( STRING / NUMBER )

36. Concurrency Control Block (Optimistic Locking + Advisory Locks — FTR-062)

Configurable at three levels with inheritance: Context level — default for all aggregates in the context Aggregate level — override for a specific aggregate Entity level — override for a specific entity Nested entities inherit from aggregate root. Aggregate roots inherit from context default. No CONCURRENCY block = off (backward compatible). Example: CONTEXT Patient @uuid "..." { CONCURRENCY { OPTIMISTIC version; ADVISORY; LOCK_TIMEOUT "5m"; } ENTITY Patient @uuid "..." IDENTIFIED_BY id (...) -- inherits OPTIMISTIC + ADVISORY from context } AGGREGATE Order @uuid "..." ROOT Order CONCURRENCY { OPTIMISTIC version; } DOC "Order with optimistic locking only";

37. Expression Grammar (for CHECK constraints, WHERE clauses)

Precedence-climbing expression parser used internally. Precedence (highest to lowest): 6: * / % 5: + - 4: < <= > >= IN NOT_IN BETWEEN IS_EMPTY INTERSECTS SUBSET_OF CONTAINS_ALL 3: = != <> == <=> LIKE MATCHES ~ 2: AND && 1: OR || ── Computation expressions (Tier 2 — ADR 20260307-functional-expression-language) ── List Comprehensions (Tier 3 — ADR 20260307-functional-expression-language) Lambda expressions are single-expression, no closures over mutable state. Bounded by collection length — preserves non-Turing-complete guarantee. ── Aggregation Built-ins (Tier 3 — ADR 20260307-functional-expression-language, FTR-299) Single-argument aggregate functions over typed collections.

Expression <- OrExpr
OrExpr <- AndExpr ( ( 'OR' / '||' ) AndExpr )*
AndExpr <- CompareExpr ( ( 'AND' / '&&' ) CompareExpr )*
CompareExpr <- RangeExpr ( CompareOp RangeExpr )*
CompareOp <- '=' / '!=' / '<>' / 'LIKE' / 'MATCHES' / '~'
RangeExpr <- AddExpr ( CollectionOp / RangeOp AddExpr )*
CollectionOp <- 'NOT' 'IN' ListLiteral
RangeOp <- '<' / '<=' / '>' / '>=' / 'BETWEEN'
ListLiteral <- '(' Expression ( ',' Expression )* ')'
ListOrExpr <- ListLiteral / AddExpr
AddExpr <- MulExpr ( ( '+' / '-' ) MulExpr )*
MulExpr <- UnaryExpr ( ( '*' / '/' / '%' ) UnaryExpr )*
UnaryExpr <- ( 'NOT' / '!' / '-' ) UnaryExpr
PrimaryExpr <- '(' Expression ')'
IfExpr <- 'IF' Expression 'THEN' Expression 'ELSE' Expression
LetExpr <- 'LET' IDENTIFIER '=' Expression 'IN' Expression
MatchExpr <- 'MATCH' Expression '{' MatchCase+ DefaultCase '}'
MatchCase <- 'CASE' Expression '=>' Expression ','?
DefaultCase <- 'DEFAULT' '=>' Expression ','?
MapExpr <- 'MAP' '(' Expression ',' IDENTIFIER '->' Expression ')'
FilterExpr <- 'FILTER' '(' Expression ',' IDENTIFIER '->' Expression ')'
ReduceExpr <- 'REDUCE' '(' Expression ',' Expression ',' IDENTIFIER IDENTIFIER '->' Expression ')'
AggregationCall <- ('SUM' / 'COUNT' / 'MIN' / 'MAX' / 'AVG') '(' Expression ')'

Example: PLUGIN AuditTrailPlugin { IMPLEMENTS Security.AuditLogger TRANSPORT SyncVirtual CIRCUIT_BREAKER { FAILURE_THRESHOLD 5 TIMEOUT "30s" BACKOFF Exponential BASE "100ms" MAX "10s" JITTER 15 ON_OPEN Warn ON_HALF_OPEN Probe ON_TRIP Emit } }

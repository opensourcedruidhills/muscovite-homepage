18. Saga

18b. State Machine (Entity Lifecycle — ADR 20260221) Context-level construct for formal intra-entity lifecycle modeling. Generates state lookup tables, transition log, C++ state machine class, gRPC transition RPCs, and Qt state-aware UI. Example: STATE_MACHINE OrderLifecycle @uuid "..." FOR Order { STATE Draft DOC "Order being composed"; STATE Submitted DOC "Order submitted for processing"; STATE Fulfilled DOC "Order shipped and delivered"; STATE Cancelled DOC "Order cancelled"; INITIAL Draft; TERMINAL Fulfilled, Cancelled; TRANSITION Draft -> Submitted ON OrderSubmitted; TRANSITION Submitted -> Fulfilled ON ShipmentDelivered GUARD "NOT has_shipped"; TRANSITION Draft -> Cancelled ON OrderCancelled; } 18c. Extension Point (Plugin System — ADR 20260223) Context-level construct declaring a typed extension contract. Code generation is driven by PLUGIN wiring in .mus (FTR-137). No generated code from EXTENSION_POINT alone — grammar + IR + validation only. Example: EXTENSION_POINT PasswordValidator @uuid "..." { CLASS: Validator; TARGET: User; LAYER: Backend; INTERFACE: SyncVirtual; DOC "Validates password strength before user creation"; }

34. Anti-Corruption Layer

35a. MAPPING Block (Brownfield Migration Factory — FTR-080, ADR 20260224-002) MAPPING blocks define field-level transformations between legacy schema structures and Muscovite target entities. Used by the migration factory generators to produce type-safe extractor and transformer components. Example: MAPPING LegacyCustomer @uuid "..." { FROM "legacy_customers" INTO Customer; FIELD customer_id -> id COPY_UUID; FIELD fname -> given_name @required COPY; FIELD lname -> family_name @required COPY; FIELD customer_email -> email @unique VALIDATE_EMAIL; FIELD birth_dt -> birth_date COERCE_DATE format="MM/dd/yyyy"; FIELD is_active -> active DEFAULT true COPY_BOOL; FIELD created_ts -> created_at COERCE_TIMESTAMP; DOC "Maps legacy customer table to Patient entity"; } 35b. FOREIGN CONTEXT (Anti-Corruption Layer — FTR-144, ADR 20260223) Typed ACL constructs for interfacing with external systems. FOREIGN_TYPE declares the shape of external data structures. TRANSLATE rules define declarative field-level mappings. The TRANSLATE sub-grammar is intentionally restricted: no IF/CASE/WHEN, no lambdas, no function calls (except type casts via ::TYPE). Complex logic belongs in Layer 2 Sidecar files. Example: FOREIGN CONTEXT PaymentGateway @uuid "..." { FOREIGN_TYPE PaymentResponse @uuid "..." ( transaction_id TEXT NOT_NULL @uuid "...", amount_cents INTEGER NOT_NULL @uuid "...", currency_code TEXT NOT_NULL @uuid "...", status_text TEXT NOT_NULL @uuid "..." ) DOC "External payment gateway response"; TRANSLATE Payment FROM PaymentResponse @uuid "..." { id <- transaction_id::UUID, amount <- amount_cents / 100, currency <- currency_code, is_approved <- status_text == "APPROVED" }; }
